# 大模型宋词生成项目总报告提纲

这份提纲面向“完整总报告”写作，不是 PPT，也不是摘要版说明。  
目标是为后续撰写一份事无巨细、几乎覆盖全项目全部内容的长篇报告提供骨架。  
这份总报告预计会包含大量表格、图表、样本展示、方法说明和附录，强调以下三点：

1. 把项目的研究目标、任务设计、数据来源、生成流程、分析流程完整讲清楚。
2. 把主要结果尽可能系统化地展开，必要时分数据库、分模型、分词牌、分主题、分指标逐一呈现。
3. 把所有关键指标的生成原理与计算逻辑讲透，做到读者即使不看代码，也能理解报告中的数据是怎样来的。

---

## 一、报告总目标与建议写法

### 1.1 这份总报告要回答什么

这份总报告建议完整回答以下问题：

1. 这个项目到底研究什么，为什么值得研究。
2. 六个 AI 数据库分别代表什么任务，真实宋词库又起什么作用。
3. 整个项目是如何从 Prompt 设计走到数据生成，再走到格律分析、主题意象分析、真实来源分析和同质化分析的。
4. 大模型写宋词时，形式能力、内容偏好、来源风险、内部同质化分别呈现出什么特点。
5. 不同模型在不同任务条件下各自擅长什么、缺陷是什么。
6. 这些结果对后续使用 AI 生成宋词、评估宋词、筛选宋词有什么方法论启发。

### 1.2 建议的写作原则

1. 先写项目框架与数据来源，再进入结果。
2. 先用中文自然语言解释概念，再引入字段名和缩写。
3. 先写总体结论，再分数据库、分模型、分词牌、分主题展开。
4. 每一节都尽量明确：这一节的问题是什么、用什么数据回答、最后得到什么结论。
5. 报告正文中放核心表格，更多细表、原始统计表和补充案例放附录。

## 五、建议的写作顺序

为了避免一开始就陷入几十张表格中，建议实际撰写顺序不是按章节自然顺序写，而是按下列顺序推进：

1. 先写第二章、第三章、第四章，把任务、数据和方法说明写扎实。
2. 再写第五章、第六章、第八章、第九章，把核心结果章节搭起来。
3. 然后写第十章案例分析，让抽象结论落到样本上。
4. 最后再写第一章、十一章、十二章、十三章，把导论、讨论、结论和局限收住。

这样会比从“绪论”一路写到最后更稳，也更不容易中途返工。
