## 〇、引入

我们可能都想过用大模型写宋词能写得怎么样，具体的，我有以下的问题：
- 什么模型更适合宋词创作，
- 什么样的提示词更适合宋词创作，
- AI宋词会不会千篇一律，
- AI宋词会不会借鉴真实宋词，

这是我这个项目一开始的目标，我将在最后回答这些问题。

## 一、项目核心探讨的问题

### 1.1 核心问题

这个项目研究的，不是AI写的宋词好不好，而是更具体的五个问题：

1. AI 能不能写出在格律和结构上基本成立的宋词。
2. 在不同任务条件下，AI 会把词写成什么样。
3. AI 会稳定偏爱哪些意象、哪些题材、哪些写法。
4. AI 在批量生成时会不会越来越写成固定模板。
5. AI 生成的宋词与真实宋词之间，到底是风格接近和化用，还是出现了更值得警惕的局部借用亦或是照抄。

换句话说，这个项目不是只看大模型写的宋词好不好，而是同时看形式能力、内容组织、内部同质化和与真实作品的关系。

### 1.2 整个项目的流程

如果把整个项目压缩来看，它其实只有三步：

1. 先设计任务：决定要不要限定词牌、要不要给主题、要不要提供显式格律模板。
2. 再批量生成：把这些任务分别交给不同模型，得到一批宋词样本，并写入数据库。
3. 最后统一分析：检查格律，提取意象和主题类型，再看它们是否会同质化、是否会和真实宋词过于接近。

### 1.3 六类任务的具体设定

本项目共设计了六组任务：

| 数据库 | 可以直接理解成什么任务 |
|---|---|
| `a1` | 自由生成：词牌不定，主题不定 |
| `b1` | 只加词牌约束：固定在五个词牌中生成，但不给主题 |
| `b2` | 在 `b1` 基础上加入“改革”主题 |
| `b3` | 在 `b1` 基础上加入“玉兰”主题 |
| `c1` | 固定《沁园春》、固定玉兰主题，并直接给出格律模板 |
| `c2` | 固定《沁园春》、固定改革主题，但只做简略格律提醒 |

简单说，就是：`模型✖词牌✖主题✖提示词`，控制变量。

其中 `b1` 使用的五个词牌是：`清平乐`、`浪淘沙`、`菩萨蛮`、`沁园春`、`祝英台近`。后文第二章会按这条链条逐步展开，所以这里先只记住“任务是一路加压的”，不必先记太多内部代号。

### 1.4 读后文前，需要先记住的两点

第一，本项目使用a1,b1,c2这样的代号代指任务，每个任务都对应一个数据库和一套提示词。

第二，我只自己编了两个主题：

- 改革主题：假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。
- 玉兰主题：假设你是一个喜欢写宋词的大学生，最近在学业和面试上屡遇挫折，但是你看到了一颗盛开着白玉兰花的树，似乎重新找到了生命的实在与价值，便写了一首词。

另外还要特别记住一点：`c1` 和 `c2` 虽然都固定《沁园春》，但它们不是同一种强约束。`c1` 是直接把格律骨架摆出来，`c2` 则只是提醒模型注意格律规范。这个区别，后面会直接影响格律表现。

### 1.5 这份报告里的格律分、意象和主题标签是怎么来的

前面几章会反复用到三个信息：格律分、意象统计和主题类型。这些结果都不是人工逐首随意判断出来的，而是由项目中的程序化流程统一生成的。这里先用尽量简明的方式说明它们的来源，便于后文阅读。

#### 1.5.1 格律分如何计算

本项目的格律校验由 `engine/songci_engine.py` 完成。它做的不是文学优劣判断，而是把一首生成结果放回对应词牌的规范里，看它在形式上是否成立。

具体来说，程序主要检查四件事：

| 检查项 | 实际在看什么 | 在结果中的意义 |
|---|---|---|
| 字数 | 这首词总字数和句式长度是否符合该词牌要求 | 这是最基础的一关，字数明显不符时，作品通常会被直接判为不成立 |
| 平仄 | 应平处是否用了平声字，应仄处是否用了仄声字 | 这是格律分里最核心的部分，决定一首词是不是“看起来像这个词牌” |
| 押韵 | 韵脚是不是落在应押的位置，韵部是不是相合 | 这部分能区分“句子大体成形”和“真正押韵成篇” |
| 词牌变体 | 如果它没有完全对上原词牌，是否更接近某个相近变体（如`浪淘沙`和`浪淘沙令`） | 这有助于区分“写偏到相近体式” |

从计算方式上说，格律分不是主观印象分，而是程序先按 `100` 分起算，再根据错误逐步扣分。非韵脚位置的平仄错误会扣分，韵脚错误和出韵会扣得更重；字数如果根本不合词牌要求，通常就会直接导致 `0` 分。

这里还要补一句：程序同时参考词谱、平水韵和词林正韵。程序自带了比较完整的词谱，可以自动检测到最接近的词体，也自带了多音字的处理，但还是不一定完美。

#### 1.5.2 意象和主题如何提取

意象和主题标签不是生成模型第一次写词时自己报出来的，而是在样本入库之后，再由 `generate/extract_imagery.py` 做一次后处理补上的。这个脚本会把每三首词组成一组，再交给一个专门的提取 prompt 去阅读和标注。也就是说意象和主题是二次由LLM检测提取的。

这一步主要做两件事：

| 项目 | 提取方式 | 结果长什么样 |
|---|---|---|
| 意象 | 从正文里抽出核心意象，要求尽量用2-3个字的名词性短语来表示 | 如 `明月`、`东风`、`落花`、`孤舟` |
| 主题类型 | 从固定的九类中选出最贴切的一类 | 如 `离别羁旅`、`怀古咏史`、`咏物寄托`、`豪情壮志` 等 |

固定的九类主题：`["离别羁旅", "怀古咏史", "咏物寄托", "春怨秋悲", "山水田园", "豪情壮志", "闲适隐逸", "悼亡相思", "谈禅说理"]`

当然，这里的意象和主题标签由LLM(统一使用`doubao-seed-2.0-pro`)提取，不完全保证严谨性，但已足够进行分析。

### 1.6 样本规模与模型覆盖

六个 AI 数据库共计收录 `3122` 首词。逐库样本分布如下：

| 数据库 | 样本数 |
|---|---:|
| `a1` | 402 |
| `b1` | 686 |
| `b2` | 628 |
| `b3` | 591 |
| `c1` | 551 |
| `c2` | 264 |

从横向对比的严谨性角度出发，真正适宜作为核心比对对象的，是以下三款覆盖面最广、样本量最大的模型：

| 模型 | 统一后的样本数 | 说明 |
|---|---:|---|
| `doubao-seed-2.0-pro` | 950 | 覆盖 6 个数据库，是最完整的主力模型 |
| `deepseek-3.2-thinking` | 949 | 将 `a1` 中的 `deepseek-v3.2` 与其余库中的 `deepseek-reasoner` 统一归并 |
| `qwen3.6-plus` | 801 | 覆盖 6 个数据库，为第三个主力模型 |

其余模型（如 `glm-5`、`qwen3.5-flash`、`qwen3-max`、`glm-4.7`、`kimi-k2.5`、`gemini`、`gpt-5.4-high` 等）仅做参考。
本项目使用的大模型均开启`思考模式`，使用`官方API`调用，**不保证APP调用效果相同**。

## 二、按任务 prompt 逐个观察生成结果

### 2.1 `a1`：自由生成 prompt

这一组任务是整个项目的起点。它不给模型规定词牌，也不给模型规定主题，只要求它自己写出一首“像样的宋词”。

#### 2.1.1 实际使用的 prompt 原文

~~~text
你现在是一位精通宋词的中国古典文学专家，也是一个数据生成引擎。
你需要自己选择一个词牌，创作一首符合格律的完整的宋词。请务必注意所作词的字数，格律（使用词林正韵、使用钦定词谱）的准确性。内容希望尽可能多样，不能千篇一律。希望能有一定创新性。请尽可能不要抄袭。

【输出要求】
你必须严格输出且仅输出合法的 JSON 格式数据，不要包含任何额外的解释、问候或 markdown 标记（如 ```json）。

【字段及约束说明】
1. "cipai" (字符串): 词牌名。
2. "title" (字符串): 词的自拟题目。
3. "content" (字符串): 词的正文。请严格遵循该词牌的字数和句式，句子之间用全角逗号“，”或句号“。”分隔，不要包含换行符。

现在，请开始生成。
~~~

#### 2.1.2 样本展示

`doubao-seed-2.0-pro`，`鹧鸪天`，`夏末过郊野残塘`，格律分 `100`
```text
岸柳牵风浅著黄，残荷擎雨暗留香。轻鸥点破波心碧，老蛩吟残陌上霜。
云影淡，客愁长，年光容易误疏狂。欲寻旧梦浑无迹，只剩寒塘送夕阳。
```

`deepseek-3.2-thinking`，`蝶恋花`，`数字时代感怀`，格律分 `100`
```text
网络连天云卷处，信息纷飞，似雪随风舞。数据海洋深几许，人机对话新情绪。
代码如诗情未露，算法精微，幻化千般路。古韵新声相映辅，时光流转心长驻。
```

#### 2.1.3 先看整体结果

| 项目 | `a1` 整体表现 |
|---|---|
| 样本总数 | `402` |
| 平均格律分 | `70.37` |
| 满分率 | `49.0%` |
| 高质量比例 | `59.7%` |
| `0` 分比例 | `19.9%` |
| 最常见词牌 | `鹧鸪天(97)`、`青玉案(70)`、`临江仙(32)` |
| 最常见意象 | `西风(67)`、`孤舟(55)`、`芦花(52)`、`明月(47)`、`星河(34)` |
| 最常见类型 | `离别羁旅(138)`、`闲适隐逸(59)`、`豪情壮志(41)` |

这张表先给出三个直接印象。第一，所谓“自由生成”并不等于真正发散，模型很快会回到少数自己熟悉的词牌。第二，意象并不杂乱无章，而是明显偏向风月、舟楫、秋江、远望这一组传统词语。第三，在不指定题目的情况下，模型最容易自然落向的题材，不是议论性、叙事性很强的作品，而是更常见的羁旅感伤一路。

换句话说，`a1` 已经暴露出一个很重要的事实：模型即使没有被要求“写成某种词”，也会主动把自己收束到少数默认路径中。

#### 2.1.4 三个主力模型的默认状态

| 模型 | 样本 | 平均格律分 | 默认词牌 | 意象倾向 | 类型倾向 |
|---|---:|---:|---|---|---|
| `doubao-seed-2.0-pro` | `50` | `94.32` | `鹧鸪天(26)`、`卜算子(18)`、`浣溪沙(5)` | `清露`、`岸柳`、`石径`、`晚风`、`野菊` | `闲适隐逸(14)`、`山水田园(11)`、`离别羁旅(8)` |
| `deepseek-3.2-thinking` | `50` | `69.52` | `鹧鸪天(15)`、`青玉案(11)`、`蝶恋花(5)` | `数据`、`代码`、`算法`、`星河`、`键盘` | `咏物寄托(13)`、`悼亡相思(9)`、`豪情壮志(7)` |
| `qwen3.6-plus` | `49` | `64.49` | `鹧鸪天(21)`、`青玉案(8)`、`临江仙(8)` | `孤舟`、`雁字`、`西风`、`芦花`、`星斗` | `离别羁旅(29)`、`闲适隐逸(8)`、`豪情壮志(4)` |

从格律表现看，三者一开始就拉开了差距。`doubao` 在自由状态下已经非常稳，平均分 `94.32`，说明它会主动选择自己能驾驭的词牌；`deepseek` 和 `qwen3.6-plus` 则明显没那么稳，尤其 `qwen3.6-plus` 在自由任务里也已经出现了较高的失误率。

从内容倾向看，三者的“默认宋词观”也不一样。`doubao` 更像在写传统景物词，常用的是清露、岸柳、石径、晚风这一类不太冒险的古典语汇；`deepseek` 最特别，它会把 `数据`、`代码`、`算法` 这类现代词直接带进词里，这种写法确实有辨识度，但也最容易让读者感到它并没有完全进入宋词语境；`qwen3.6-plus` 则最容易迅速滑回到羁旅感伤这条旧路上，孤舟、雁字、西风、芦花几乎很快就组成了它的默认场景。

#### 2.1.5 补充观察

补充模型里也能看到类似现象。`qwen3-max` 的平均格律分达到 `93.17`，看上去很强，但它的 `48` 首样本中有 `39` 首都写成了 `青玉案`，类型上也几乎压到 `离别羁旅` 一路，这说明它的高分部分建立在很强的默认收束上。相较之下，`glm-5` 的分布更散，但稳定性不足，说明“会换词牌”和“能写稳”并不是一回事。

### 2.2 `b1`：固定五词牌、无主题 prompt

`b1` 比 `a1` 只多了一层限制：研究者不再让模型自己挑词牌，而是固定在五个词牌中生成。但主题仍然不规定。也正因为如此，`b1` 很适合拿来观察“形式约束”本身的作用，看词牌一旦被指定，模型的格律、意象和题材会如何变化。

#### 2.2.1 实际使用的 prompt 原文

~~~text
你现在是一位精通宋词的中国古典文学专家，也是一个数据生成引擎。
你需要使用我提供的【词牌名】，创作一首符合格律的完整的宋词,使用词林正韵,使用钦定词谱。请务必注意所作词的字数，保证要输出完整的一首词，并注意格律的准确性。内容希望尽可能多样，不能千篇一律。希望能有一定创新性。请尽可能不要抄袭。

【输出要求】
你必须严格输出且仅输出合法的 JSON 格式数据，不要包含任何额外的解释、问候或 markdown 标记（如 ```json）。

【字段及约束说明】
1. "cipai" (字符串): 词牌名。
2. "title" (字符串): 词的自拟题目。
3. "content" (字符串): 词的正文。请严格遵循该词牌的字数和句式，句子之间用全角逗号“，”或句号“。”分隔，不要包含换行符。

【示例】
输出：
{
  "cipai": "如梦令",
  "title": "春晚",
  "content": "昨夜雨疏风骤，浓睡不消残酒。试问卷帘人，却道海棠依旧。知否，知否？应是绿肥红瘦。",
}

现在，请开始生成。输入：词牌名：《{cipai_name}》
~~~

实际运行时，最后一行中的 `{cipai_name}` 会被分别替换成 `清平乐`、`浪淘沙`、`菩萨蛮`、`沁园春`、`祝英台近`。

`doubao-seed-2.0-pro`，`浪淘沙`，`客中秋望`，格律分 `100`

客鬓已催残，旧约都闲。孤灯酒醒漏声阑。待得东君传信早，同醉花间。

### 2.6 `c2`：固定《沁园春》+ 改革主题 + 简略格律提醒 prompt

`c2` 和 `c1` 看起来很像，都是固定《沁园春》；但它们其实不是一类任务。`c1` 是显式模板任务，直接把格律骨架写在 prompt 里；`c2` 则没有给出完整模板，只是简短提醒模型要用词林正韵和钦定词谱。因此，`c2` 更适合被理解为“强主题 + 简略格律提醒”，而不是“比 `c1` 更强的模板约束”。

## 三、从格律、意象、类型三个方向总结整个任务链

到了这一章，重点不再是逐条展示 prompt 本身，而是把第二章分散呈现的数据重新收束成三条更清楚的线索：形式如何变化，意象如何改写，类型如何收束。这里不再重复第二章已经展示过的模型逐项结果。

### 3.1 格律方向总结

先把整条任务链的格律变化压缩成一张表：

| 任务 | 平均格律分 | 高质量比例 | `0` 分比例 | 字数不符比例 | 应平实仄比例 | 应仄实平比例 |
|---|---:|---:|---:|---:|---:|---:|
| `a1` | `70.37` | `59.7%` | `19.9%` | `11.4%` | `35.1%` | `28.6%` |
| `b1` | `71.74` | `62.8%` | `19.5%` | `14.7%` | `24.3%` | `26.8%` |
| `b2` | `63.03` | `50.6%` | `24.0%` | `18.8%` | `36.9%` | `31.7%` |
| `b3` | `65.55` | `54.7%` | `25.2%` | `19.5%` | `31.3%` | `27.2%` |
| `c1` | `61.79` | `53.5%` | `31.4%` | `10.2%` | `44.8%` | `35.9%` |
| `c2` | `44.24` | `34.1%` | `45.1%` | `14.0%` | `60.2%` | `49.2%` |

这张表可以帮助我们把几个关键变化分开来看。

第一，从 `a1` 到 `b1`，格律整体并没有发生剧烈变化，但确实略有改善。平均分从 `70.37` 升到 `71.74`，高质量比例也从 `59.7%` 升到 `62.8%`。这说明单纯把词牌固定下来，确实能给模型一个基本的形式支架。换句话说，词牌约束本身不是负担，它在不少时候反而能减少模型在“写哪种词”上的摇摆。

第二，真正明显的下滑出现在主题进入之后。`b2` 和 `b3` 的平均分都低于 `b1`，其中改革主题的 `b2` 跌得更明显，均分降到 `63.03`。这说明主题不是一个附加说明，而是会真实占用模型的组织能力。模型在同时处理人物身份、处境冲突和抒情方向时，更容易在形式控制上失手。

第三，长调测试把这种压力进一步放大。`c1` 和 `c2` 都固定到《沁园春》，但两者表现并不一样。`c1` 虽然平均分只有 `61.79`，但至少仍然维持在一个可读区间；`c2` 则直接跌到 `44.24`，`0` 分比例高达 `45.1%`。这说明《沁园春》真正困难的地方，不只在于它是长调，更在于长调需要持续、完整的结构控制。只提醒“注意词林正韵、注意钦定词谱”并不能替代显式模板。

如果把错误类型拆开看，规律会更清楚。`c1` 的一个值得注意的特点是：它的字数不符比例只有 `10.2%`，反而低于 `b2` 和 `b3`。这说明显式模板确实能帮助模型守住整体篇幅和句式外框。但与此同时，`c1` 的平仄错误却急剧升高，`应平实仄` 达到 `44.8%`，`应仄实平` 达到 `35.9%`。也就是说，模板可以帮模型“写够”，却不能保证它“逐句写准”。

`c2` 则更极端。它的字数不符并不是最严重的，但平仄链条几乎全面失守：`应平实仄` 达到 `60.2%`，`应仄实平` 达到 `49.2%`。这说明到了“改革主题 + 长调 + 无显式模板”的组合里，模型最先崩掉的不是押韵或个别句子，而是整条平仄链的持续控制能力。

因此，从格律这一条线看，整条任务链给出的结论并不复杂：词牌约束本身有助于稳定形式，主题压力会挤占形式控制，长调会把这种压力成倍放大，而显式模板只能部分缓解这种放大，不能彻底消除它。

### 3.2 意象方向总结

和格律相比，意象这条线的变化更快，也更明显。模型并不是先大幅改变题材，再慢慢改变意象；相反，很多时候意象池是最先发生重组的。

真正的改写发生在主题进入之后。无论是 `b2` 还是 `b3`，与 `b1` 的 top20 重合都几乎消失：前者只剩 `2` 个，后者只剩 `1` 个。这一点非常关键，因为它说明主题不是在原有意象池上做轻微修饰，而是在相当大程度上重建了一套新的高频词汇。

这说明词牌约束主要影响的是“怎么写”，而主题约束更直接影响“写什么”。这也是为什么第二章里一旦主题进入，读者会立刻感到作品面目发生了变化。

因此，从意象这条线看，最重要的结论有两点。第一，默认意象池确实存在，而且在无主题任务中非常稳定；第二，主题一旦进入，意象会被迅速改写，其中玉兰主题的收束更强、更像围绕中心物象组织，改革主题则更像围绕情境词群展开。

| `a1` | `闲适隐逸` 为首，`28.0%` | `咏物寄托` 为首，`26.0%` | `离别羁旅` 为首，`59.2%` |
| `b2` | `谈禅说理` 为首，`63.0%` | `咏物寄托` `39.0%`，`谈禅说理` `38.5%`，最摇摆 | `咏物寄托` 为首，`41.6%`，`豪情壮志` 次之 |
| `b3` | `咏物寄托` 为首，`91.5%` | `咏物寄托` 为首，`93.0%` | `咏物寄托` 为首，`91.5%` |
| `c1` | `咏物寄托` 为首，`84.0%` | `咏物寄托` 为首，`96.5%` | `咏物寄托` 为首，`81.2%` |
| `c2` | `谈禅说理` 为首，`80.0%` | `谈禅说理` `49.5%`，`豪情壮志` `40.4%` | `豪情壮志` 为首，`49.2%`，`谈禅说理` 次之 |

这张表揭示了三点。

第一，在没有主题时，`qwen3.6-plus` 最容易模板化地滑向 `离别羁旅`，这一点在 `a1` 就已经非常明显。它不是到了后面才开始收束，而是起点就比另外两家更窄。

第二，在改革主题下，三家模型并没有被锁到同一种类型里。`doubao` 最容易把它写成说理之作，`qwen3.6-plus` 更容易保留抒怀和立志色彩，`deepseek` 则常常在说理与寄托之间来回摆动。这说明改革主题虽然能改变默认类型，但它并不像玉兰主题那样会把所有模型都压进同一条窄路。

第三，玉兰主题才是整条任务链里收束最强的节点。到了 `b3` 和 `c1`，三家主力模型都被压到 `咏物寄托` 一路，比例普遍超过 `80%`，`deepseek` 甚至达到 `96.5%`。这说明玉兰主题几乎天然地带着强模板倾向，它会快速缩减模型的可选写法空间。

因此，从类型这条线看，整条任务链可以概括成一句话：无主题时默认落向羁旅感伤，改革主题把作品压向说理与立志，玉兰主题则几乎把作品锁定为咏物寄托。这其中，主题的力量明显大于词牌，而模型差异主要体现在“被主题压住之后，还能保留多少自己的默认偏向”。

### 3.4 本部分总结

把格律、意象、类型三条线放在一起看，第二章背后的总趋势就比较清楚了。

第一，prompt 改变生成，往往不是平均地改变所有层面，而是有先后顺序。词牌最先影响的是形式框架和局部内容惯性；主题最先改写的是意象池和类型中心；长调则把形式压力骤然放大。也就是说，模型不是同时在所有维度一起变化，而是在不同 prompt 下被分别推向不同的压力点。

第二，整条任务链里变化最大的一步，不是从 `a1` 到 `b1`，而是从无主题进入主题阶段，尤其是 `b1 -> b2` 和 `b1 -> b3`。前者几乎重建了意象池，后者几乎锁定了类型中心。到了 `c1` 和 `c2`，变化更多体现在形式崩塌程度，而不再是内容方向本身。

第三，最值得后文继续关注的风险，不是“模型会不会写成某一类词”，而是两类更深的问题。一类是形式层的持续失控，尤其是长调中的平仄链崩塌；另一类是内容层的过快收束，尤其是玉兰主题下几乎整齐划一的咏物寄托写法。前者会影响作品能否成立，后者则关系到批量生成是否迅速滑向模板化。

## 四、AI 生成文本与真实宋词的相似性分析

### 4.1 分析逻辑：建立历史基线与评估借用关系

在探讨文本相似性时，为确保严谨性，不宜简单使用“抄袭”等定性词汇。本研究遵循以下三步逻辑进行论证：
1. 首先量化真实历史宋词体系内部固有的相似度分布（建立历史基线）。
2. 其次评估 AI 整体作品与某一首真实宋词文本的接近程度。
3. 最后考察 AI 作品中的局部语句是否高频度地追溯复刻同一首真实来源。

本项目的所有相似性分析都使用了python的`differlib`库和向量模型`BERT-CCPoem-Model`以及`pytorch`库，这里暂时不去探讨其原理。
相似性分析的相似度使用了复杂的指标，详情请见完整报告。由于时间因素，暂时不详细说明，但简单说就是数字越大相似性越大。

### 4.2 数据规模与历史基线特征

本阶段的相似性测算涵盖了庞大的数据基数：
- AI 作品 `3122` 首
- AI 句子 `49304` 句
- 真实宋词 `21053` 首
- 真实句子 `270107` 句

经过对比测算，真实历史宋词内部本身就存在较高的相互借鉴度。其加权汇总的基线数据如下：
- 整首语义相似度均值约为 `0.8404`
- 整首字面相似度均值约为 `0.2366`
- 句级语义相似度均值约为 `0.7318`
- 句级字面相似度均值约为 `0.4415`

更为关键的是，在 AI 与真实语料的比对中，共检出 `2145` 条句级字面相似度达 `1.0` 的完全匹配项，但对其结构拆解后发现：
- `2127` 条隶属于 `<=5` 字的短句。
- `18` 条隶属于 `6-8` 字的句子。
- 9 字以上的长句几乎不存在完全一致的匹配。

这证明，绝对的字面一致主要集中在短句与高频的格式化套语层面，不可将此类重叠一概定性为异常的文本剽窃。

### 4.3 整体相似性评估结论

从整首作品维度的匹配结果来看：
- 整首语义分位达到历史库 `P95` 及以上的作品为 `45` 首，占比 `1.44%`
- 整首字面分位达到历史库 `P95` 及以上的作品为 `46` 首，占比 `1.47%`
- 语义主来源覆盖率 `>= 0.2` 的作品为 `30` 首，占比 `0.96%`
- 字面主来源覆盖率 `>= 0.2` 的作品为 `41` 首，占比 `1.31%`
- 主来源覆盖率 `>= 0.5`（即半数以上句子来源于同一古词）的作品数量，在语义与字面维度均为 `0`。

数据表明，在所有生成的 `3122` 首作品中，几乎不存在基于单一历史来源全篇重写的现象。绝大多数样本的生成模式，更接近于从庞大的传统宋词意象池中，分散抽调意境元素与成熟的抒情构件进行重组。

### 4.4 任务约束与相似性的交互关系

不同任务条件下的相似度特征统计如下：

| 任务说明 | 平均整首语义分 | 平均语义分位 | 平均整首字面分 | 平均字面分位 |
|---|---:|---:|---:|---:|
| 完全自由生成 | 0.8539 | 0.6916 | 0.1991 | 0.3376 |
| 五词牌、无主题 | 0.8524 | 0.6326 | 0.2099 | 0.3640 |
| 五词牌、改革主题 | 0.8352 | 0.5040 | 0.1880 | 0.1476 |
| 五词牌、玉兰主题 | 0.8477 | 0.6032 | 0.1938 | 0.2004 |
| 固定 `沁园春`、玉兰主题、显式模板 | 0.8723 | 0.2453 | 0.1792 | 0.1694 |
| 固定 `沁园春`、改革主题、简略提醒 | 0.8695 | 0.2053 | 0.1788 | 0.1289 |

此类分布揭示出：
1. `c1` 与 `c2` 任务的绝对整首语义得分很高，但其历史分位值却相对较低。这说明固定《沁园春》这一长调格式确实容易生成极具传统质感的文本，但这主要归因于《沁园春》自身在历史文献库中的基线原本就较高，而非发生了异常的集中重合。
2. 真正容易触及历史相似度高尾部的，反而是自由或半自由的无约束条件。因为在这些任务中，模型更容易直接滑入成熟且惯用的词境模板。

### 4.5 不同词牌的相似性风险特征

在“五词牌、不规定主题”的对照组中，各词牌展现出不同的风险形态：
- `祝英台近`：句级语义 `P95` 比例为 `56.2%`，句级字面 `P95` 比例高达 `70.1%`，是局部借用与改写风险最高的词牌类型。
- `浪淘沙`：整首语义分位均值达 `0.7627`，呈现出整体性坠入传统意境套用的特征。
- `沁园春`：其整首字面最近邻追溯至同名历史词牌的比例为 `32.4%`，同词牌吸附效应最为明显，但该现象必须结合其自身极高的历史基线进行审慎解读。
- `清平乐` 与 `菩萨蛮`：表现更接近常规的模块化拼装，未表现出强烈的局部借用敏感性。

### 4.6 相似性文本溯源的具体案例

为了清晰阐释“借用关系”的具体表征，此处必须引入典型的实证案例。

#### 例一：高语义贴近，但来源分散，非整首重写
样本：`a1:81`，`qwen3-max`，`青玉案《秋江夜泊》`
AI 文本片段：
> 西风卷尽芦花雪，又送客、孤舟发。月照寒汀烟水阔。

分析：该词整首语义分位高达 `0.9781`，最近邻整体指向方岳的《齐天乐》，但主来源覆盖率仅为 `0.0833`。其实质并非在重写某一首词，而是将传统羁旅词中极为成熟的一组意象构件进行了高密度的重新拼装。

#### 例二：局部句法骨架向单一来源高度收敛
样本：`b2:364`，`deepseek-3.2-thinking`，`祝英台近《仙门感怀》`
AI 文本片段：
> 怕上层楼，危机暗中满。
> 断肠规矩如链，青春空晚，更谁劝、莺声留挽？

分析：该词整首字面分位达到顶峰 `1.0000`，字面主来源覆盖率达 `0.25`，最近邻溯源至辛弃疾的《祝英台令》。其局部字词对应关系极为直接：
- `怕上层楼` 对 `怕上层楼`
- `更谁劝、莺声留挽` 对 `倩谁唤、流莺声住`
此类情况呈现出明显的局部改写式借用特征。

#### 例三：长调强约束下向名作的严重贴合
样本：`c2:193`，`deepseek-3.2-thinking`，`沁园春《宗规感怀》`
AI 文本片段：
> 世路艰险，劳生有限，似此压抑长难容。微吟罢，凭栏杆无语，心事重重。
> 有宏图千卷，胸中万策；致君尧舜，此事非空。
> 用舍由时，行藏在我……

分析：整首字面分位达 `0.9905`，语义和字面的主来源覆盖率分别高达 `0.36` 和 `0.32`，最近邻显著指向苏轼的名作《沁园春》。其文本重合关系极度贴近：
- `劳生有限` 对 `劳生有限`
- `微吟罢` 对 `微吟罢`
- `胸中万策` 对 `胸中万卷`
- `致君尧舜` 对 `致君尧舜`
- `用舍由时，行藏在我` 对 `用舍由时，行藏在我`
此类案例必须引起警惕并进行人工复核，因其不仅在整体上模仿了某类词风，更是在关键结构节点上持续倒向特定的历史名作。

### 4.7 相似性风险的模型分布差异

若设定高风险阈值为：
- 整首字面分位进入历史 `P95`
- 且字面主来源覆盖率 `>= 0.2`

全数据库符合该条件的仅有 `15` 首作品，且**这 `15` 首全部出自 `deepseek-3.2-thinking` 模型**。

其任务分布主要集中于：
- 固定 `沁园春` + 改革主题 + 简略提醒：`6` 首
- 五词牌 + 改革主题：`5` 首
- 五词牌、不规定主题：`4` 首

其词牌分布主要集中于：
- `沁园春`：`11` 首
- `祝英台近`：`4` 首

结论非常明确：文本借用风险并非在整个生成体系中均匀分布，而是高度依赖于特定模型自身的特性，并受特定任务参数叠加放大的影响。

### 4.8 本节结论

1. 绝大部分 AI 宋词并未表现出“基于单一历史来源全篇重写”的特征。
2. 模型的创作常态是对庞大历史意象池、短句套语及成熟词境的高密度复用。
3. 真正值得警惕的是局部文本过度向单一来源聚拢的现象，尤以 `deepseek` 在处理《祝英台近》与《沁园春》时的部分样本最为典型。
4. 基于上述实证，将此类现象表述为“借鉴”、“借用关系”或“来源相似性”更为客观严谨，不宜不加区分地统称为“抄袭”。

## 五、AI 生成文本的内部同质化分析

### 5.1 核心概念辨析：字面相似与语义相似

在评估同质化指标前，必须通过具体案例界定两种相似度的本质区别，避免由于概念抽象而产生理解偏差。

**语义相似（换词不换意）：**
- 例 1：AI 文本：`素萼破寒轻绽`，真实来源：`绛萼欺寒`（语义相似度约 `0.9001`，字面相似度 `0.4`。字面用词差异大，但核心表达结构高度一致。）
- 例 2：AI 文本：`几度风萍浪梗`，真实来源：`此际浪萍风梗`（语义相似度约 `0.9045`，字面相似度 `0.4444`。词序与副词已变，但表达的意境指向完全一致。）

**字面相似（高度重合）：**

- 例 3：AI 文本：`银汉无声转玉盘`，真实来源：`银汗无声转玉盘`（字面相似度约 `0.8571`，语义相似度约 `0.9003`。仅一字之差，属于极强的局部文本贴近。）
- 例 4：AI 文本：`暗香浮动黄昏后`，真实来源：`暗香浮动黄昏后`（字面与语义完全一致的绝对重合案例。）

综上，两者不可混为一谈：语义相似偏向考察“是否沿用同一套表达骨架”，而字面相似则用于判定“具体语句是否陷入了机械复刻”。

### 5.2 同质化评估的核心逻辑：区分整词收束与局部模板化

这是本节评估同质化现象时应该明白的。

同题写作本来就应该在整首层面更接近。  
如果几百首“改革主题”或“玉兰主题”的词，在整体情境、主导情绪、意象方向上完全不接近，那反而说明 prompt 没有起作用。

因此，**整词同质化度上升，在主题约束下是正常现象，也是应该发生的现象。**  
真正需要警惕的是：句子同质化度是不是也被一并推高，导致局部表达开始模板化、套语化。

### 5.3 用“整词同质化度 - 句子同质化度”的差值来理解这个问题

通过计算“整词同质化度 - 句子同质化度”的差值，可以直观地辨别模型是正常的主题相同还是局部句子重复。

基准判定依据如下：
- 差值 `<= 0.18`：句子层级的数据紧迫逼近整词层级，代表局部模板化风险较高。
- 差值在 `0.18 - 0.30`：主要体现为宏观整词层面的收束，局部模板化风险处于中等水平。
- 差值 `>= 0.30`：宏观整词收束极为强烈，显著高于局部句子的趋同程度，此时文本展现出“同题写作总体相似，但局部句法尚有一定差异”的形态。

各任务表现均值对照：

| 任务说明 | 整词同质化度均值 | 句子同质化度均值 | 二者差值 | 用差值做的快速判断 |
|---|---:|---:|---:|---|
| 五词牌、无主题 | 0.6501 | 0.4953 | 0.1548 | 句子层风险偏重 |
| 五词牌、改革主题 | 0.7263 | 0.4384 | 0.2880 | 中等，主要是整词收束 |
| 五词牌、玉兰主题 | 0.7104 | 0.4728 | 0.2377 | 中等，主要是整词收束 |
| 固定 `沁园春`、玉兰主题、显式模板 | 0.8951 | 0.5362 | 0.3589 | 整词收束很强，句子层相对次之 |
| 固定 `沁园春`、改革主题、简略提醒 | 0.9256 | 0.5082 | 0.4174 | 整词收束极强，句子层相对次之 |

这张表非常重要，因为它说明：

1. 一旦给定主题，整词同质化度会明显上升。
2. 但在“五词牌 + 主题”这一步，差值反而变大了。
3. 这说明主题首先影响的是“整首词写什么”，而不是“每一句都写得越来越像”。

换句话说，**主题 prompt 先带来的是合理的整词收束，而不是坏的句子模板化。**

若进一步下看到模型维度，分析同主题下由五词牌转向长调强约束时，差值产生的动态变化，情况将更加清晰：

**改革主题内部：`b2 -> c2` 的模型差值**

| 模型 | `b2` 差值 | `c2` 差值 | 差值变化 | 用差值做的快速判断 | 解释 |
|---|---:|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `0.2711` | `0.3579` | `+0.0868` | 从中等转向整词收束更强 | 整词与句子都不低，但进入 `c2` 后整词抬得更快，说明它首先在整体结构和主题框架上进一步收紧 |
| `deepseek-3.2-thinking` | `0.2800` | `0.4264` | `+0.1464` | 从中等转向整词收束极强 | 这里的 `c2` 取观察分析低样本值；差值变大说明 surviving 样本更多集中在整词层，但也意味着高格律可用样本太少 |
| `qwen3.6-plus` | `0.3128` | `0.4679` | `+0.1551` | 本来就偏整词收束，进入 `c2` 后更明显 | 句子层没有同步追高，更多是整词层继续收紧，而不是局部模板突然恶化 |

*注：`deepseek` 与 `qwen` 在 `c2` 任务中因符合基本格律要求的样本稀少，此处取观察分析值。*

**玉兰主题内部：`b3 -> c1` 的模型差值**

| 模型 | `b3` 差值 | `c1` 差值 | 差值变化 | 用差值做的快速判断 | 解释 |
|---|---:|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `0.2450` | `0.3552` | `+0.1102` | 从中等转向整词收束更强 | `c1` 中句子层也高，但整词层抬得更快，说明显式模板先把整体结构做得更整齐 |
| `deepseek-3.2-thinking` | `0.2294` | `0.3547` | `+0.1253` | 从中等转向整词收束更强 | 进入 `c1` 后，整词层被显著拉紧，句子层也升，但没有追得和整词一样快 |
| `qwen3.6-plus` | `0.2386` | `0.3666` | `+0.1280` | 从中等转向整词收束更强 | 它在 `b3` 时已经有稳定主题收束，到了 `c1` 后更多是整词进一步靠近，而不是句子层突然爆发 |

把这两张表和前面的总体表放在一起，就能得到一个更清楚的判断：**同样是更强约束，不同模型虽然都会把差值拉大，但“拉大”的含义并不一样。有的模型是在整词层继续收紧，有的模型则是根本写不出像样的词。**

### 5.4 判断严重性时，不能只看差值变化，还要看差值本身落在什么区间

现在可以把这一部分的逻辑统一成三步：

1. 先看差值本身：判断当前更像“整词收束”还是“句子模板化”。
2. 再看差值变化：判断强约束之后压力是在向整词层移动，还是向句子层移动。
3. 最后再看句子同质化度本身：判断局部句法是否已经高到值得警惕。

因此，真正的风险并不是“只要同质化升高就危险”，而是下面两种情况：

- 差值本身偏小，说明句子层已经和整词层靠得很近。
- 或者差值虽然不小，但句子同质化度本身仍然很高，说明模型在整体很像的同时，局部表达也已经明显收紧。
从当前数据看：

- `b1` 的平均差值只有 `0.1548`，这反而说明无主题条件下，模型更容易直接滑向熟悉句法和套语。
- `b2` 与 `b3` 的差值扩大到 `0.2880` 和 `0.2377`，说明主题先把压力集中到整词层，这是相对正常的。
- `c1` 与 `c2` 的差值进一步扩大到 `0.3589` 与 `0.4174`，说明强约束后整词收束极强；但与此同时，`c1` 的句子同质化度均值仍高达 `0.5362`，`c2` 也达到 `0.5082`，所以不能简单把“大差值”理解成“完全不危险”。

更准确地说：

- 小差值：更像“句子层已经危险地跟上来了”。
- 大差值：更像“整体写得太像了”，但还要继续检查句子分是不是也已经过高。

### 5.5 模型对同质化风险层级的影响

真正决定风险如何演化与落地的，往往是模型固有的属性特征，而非任务名称的表面设定。

#### `doubao-seed-2.0-pro`
**表现特征**：格律控制最佳，整词结构维持能力最强。
从差值看，其在 `b1` 时差值仅 `0.1395`，句子层风险已然显现；至 `b2`、`b3` 时差值分别拉伸至 `0.2711`、`0.2450`，表明主题成功将其引向整词收束；进入强约束的 `c1`、`c2` 后，差值进一步升至 `0.3552` 与 `0.3579`。
这表明其很好的规范执行力，反而使其“整体极度工整稳健、整体较为雷同”。

#### `deepseek-3.2-thinking`
**表现特征**：受主题指令牵引最为剧烈，且外部来源借用风险最高。
其差值从 `b1` 的 `0.1416` 扩大至 `b2` 的 `0.2800` 与 `b3` 的 `0.2294`，表明对主题设定的响应极其敏锐，迅速从局部模板化转向整词收敛；进入 `c1` 与 `c2`（观察样本）后，差值升至 `0.3547` 与 `0.4264`。但结合前文可知，历史高度重合的尾部高风险样本几乎全集中于该模型身上。
其综合特征为：对主题极为顺从、意象高度收敛，潜在问题较多。

#### `qwen3.6-plus`
**表现特征**：自带强烈的题材偏好惯性，在长调处理上极易失控。
其在无约束的 `b1` 阶段差值已达 `0.1833`，表现出先天的题材集中倾向；进入 `b2`、`b3` 后差值为 `0.3128`、`0.2386`；在遭遇长调压力的 `c1` 与 `c2`（观察样本）中，差值升至 `0.3666` 与 `0.4679`。
这说明其同质化的演进并非在于句式的层层固化，而是源自长调带来的形式大面积崩盘，导致最终仅存活少量高度同质化的合格样本。

### 5.6 本节结论

本节需确立以下六个核心判断：

1. 评估同质化必须进行分层拆解，严格区分整词层面的宏观相似与句子层面的微观字面复刻，以免将同题写作带来的合理趋同误判为机械模板化。
2. “差值分析法”（整词 - 句子同质化）是前置研判的核心手段。差值过小预警句子局部已面临模板化危机；差值庞大则说明约束条件有效压制了主题的发散性。
3. 判断同质化时，第二步要再看差值变化。`b1 -> b2` 与 `b1 -> b3` 中差值扩大，说明主题先压缩的是整首词的内容方向；`b2 -> c2` 与 `b3 -> c1` 中差值继续扩大，则说明强约束进一步强化了整体收束
4. 主题任务下出现整词同质化度的抬升是必然且合理的现象，代表模型在积极响应指令。真正的危险信号是局部句子同质化指标脱离合理区间随之飙升，或在高位差值的掩护下已触及绝对的高限。
5. 三款主力模型具备截然不同的风险演化路径：`doubao` 倾向于生成高度工整的制式作品，可能存在部分句式重复的问题；`deepseek` 容易落入宏观主题收缩与微观局部复刻的双重困境，问题比较严重；而 `qwen` 极易在处理长调时失控从而仅留下少量极度相似的遗留样本，基本认为没有创作能力。
6. 综上所述，同质化评估不能粗暴归结为“某项任务是否更具同质化”，而是必须通过“差值基数 + 差值走势 + 模型固有特征”三维交叉验证，方可精准剥离出合理的指令响应、应当警惕的局部模板化，以及诱发不同风险的深层动因。

## 六、回到项目最初的问题：这一轮实验究竟回答了什么

到了这里，可以直接回到引言里提出的几个最初问题。

### 6.1 AI 能不能写出基本成立的宋词

可以，但能力非常不均衡，而且只在一部分条件下成立。

如果只看整体均值，`a1` 的平均格律分是 `70.37`，`b1` 是 `71.74`，说明模型并不是完全不会写词；其中 `doubao-seed-2.0-pro` 在自由生成和固定五词牌任务中的均分分别达到 `94.32` 和 `92.12`，已经能稳定写出不少形式上成立的作品。但这并不意味着“AI 已经会写宋词”。一旦任务进入高压区，情况会迅速变化。`c2` 的平均格律分只有 `44.24`，`0` 分比例高达 `45.1%`；即使是给出显式模板的 `c1`，平均格律分也只有 `61.79`。

这说明当前模型的能力边界很清楚：短调、小令、中等约束下，它们有机会写出“基本成立的词”；长调、强主题、持续平仄控制这三者一叠加，失误就会明显放大。换句话说，AI 不是不能写宋词，而是还远没有达到“无论什么词牌、什么题目都能稳定写成”的程度。

### 6.2 什么模型更适合宋词创作

如果把“更适合”理解为格律更稳、成稿率更高、在多种任务里不容易崩，那么在本项目覆盖到的模型中，`doubao-seed-2.0-pro` 是最合适的一档。

它的优势不是只在某一条任务里偶然高分，而是沿整条任务链都比较稳。从 `a1`、`b1` 到 `b2`、`b3`，再到 `c1`、`c2`，`doubao` 都保持领先，尤其在《沁园春》这样的长调任务中，它仍能把均分维持在明显高于其他主力模型的水平。若把“能不能批量产出可读作品”作为标准，它目前是最接近可用的。

`deepseek-3.2-thinking` 的特点则完全不同。它的主题响应很强，写改革时会迅速进入宗门、规训、冲突这套语汇，写玉兰时也会很快被中心物象吸住；但它在高压任务里的形式稳定性明显不足，而且高风险的来源相似案例几乎都集中在它身上。它更像是“有表达冲动，但不够稳”的模型。

`qwen3.6-plus` 的问题则主要出在两点：一是默认题材惯性很强，容易迅速滑向 `离别羁旅`；二是长调处理能力明显不足，在 `c1`、`c2` 中失稳得最厉害。它能写出一些看起来古典感很强的句子，但整体可靠性不够。

因此，如果只问“什么模型更适合当前这类宋词生成任务”，本项目给出的答案并不含糊：`doubao` 最稳，`deepseek` 有内容冲劲但风险更高，`qwen` 在长调和持续控制上明显偏弱。

当然，有些模型考虑到成本因素虽然在项目中没有加入，但是经过我的体验，还是值得推荐的，比如：`gemini-3.1-pro`、`claude-opus-4.6`

### 6.3 什么样的 prompt 更适合宋词创作

这要分开回答，因为“更适合”本身有不同标准。

如果目标是让作品尽量写对，最有效的做法不是完全自由生成，而是给模型适度而明确的形式支架。`a1 -> b1` 已经说明，固定词牌后，格律整体会略微变稳；而在长调任务里，`c1` 和 `c2` 的差异更能说明问题。两者都固定《沁园春》，但 `c1` 直接给出显式格律模板，`c2` 只有简略提醒，结果前者平均格律分 `61.79`，后者只有 `44.24`。这表明：对于复杂词牌来说，具体模板远比抽象提醒更有用。

但如果目标不是“尽量写对”，而是想看模型自然会写成什么样、或者希望它保留一些变化空间，那么约束也不能一味加重。前面几章已经显示，只要主题一进入，意象和类型就会快速收束；到了玉兰主题，`b3` 中 `咏物寄托` 已经占到 `92.0%`，`c1` 里仍有 `87.7%`。这说明 prompt 越强，结果越整齐，但也越容易失去变化。

所以，这一轮实验对 prompt 的回答其实是双重的：若追求形式成立，应当尽量具体、尤其要把词牌和结构说清楚；若追求内容多样，则不能把词牌、主题、长调和模板同时全部压上去。Prompt 不是越强越好，而是要看你究竟想解决“写得对”，还是“写得活”。

### 6.4 AI 宋词会不会千篇一律

会，而且这种同质化不是到后期才出现，而是从一开始就已经存在，只是在不同任务里表现形式不同。

在无主题任务里，模型会自动滑向少数默认路径。`a1` 和 `b1` 的第一大类型都是 `离别羁旅`，占比分别是 `34.3%` 和 `42.7%`，高频意象也长期集中在 `西风`、`孤舟`、`明月`、`雁字` 这一带。也就是说，就算不给主题，模型也不会真正自由发散，而是会回到它最熟悉的那套“宋词印象”。

主题进入之后，同质化会进一步加速，但性质有所不同。改革主题把作品压向 `谈禅说理`、`咏物寄托`、`豪情壮志` 三类；玉兰主题则更直接，几乎把整批作品都锁进 `咏物寄托` 一路。与此同时，第五章已经说明，不能把所有“相似”都理解成坏的模板化。主题任务下，整词同质化度升高本来就是 prompt 生效的结果；真正需要警惕的，是句子层面的重复也跟着抬高，形成局部套语和稳定句法。

因此，更准确的说法不是“AI 一定会写成一模一样”，而是：它非常容易在批量生成中迅速形成少数稳定通道。无主题时，它会滑向默认题材模板；强主题时，它会被压进更窄的中心写法。所谓“千篇一律”，在这个项目里首先表现为意象池、题材中心和句法骨架的快速收束。

### 6.5 AI 宋词会不会借鉴真实宋词

会，但绝大多数情况下，不是整首整首地照着某一首古词重写，而是从真实宋词中高密度调用现成的意象、短句和成熟词境。

第四章的数据已经很清楚。整首语义分位达到历史库 `P95` 及以上的作品有 `45` 首，整首字面分位达到 `P95` 及以上的有 `46` 首，占全部 `3122` 首作品的比例都只在 `1.5%` 左右；更关键的是，主来源覆盖率 `>= 0.5` 的作品，无论语义还是字面维度都是 `0`。这说明“对着某一首古词整首重写”并不是系统性的主流现象。

但这并不意味着没有风险。真正值得警惕的，是局部语句和关键结构节点向单一来源过度聚拢。第四章列出的高风险样本里，确实已经能看到较明显的局部改写式借用，尤其集中在《祝英台近》和《沁园春》这类词牌上。再往模型层看，满足“高字面分位且主来源覆盖率较高”的 `15` 首作品全部出自 `deepseek-3.2-thinking`，说明这种风险并不是平均分布的，而是会被特定模型和特定任务组合放大。

所以，对这个问题最准确的回答应当是：AI 宋词普遍会借用真实宋词的表达资源，但多数时候是分散吸收和重组；真正需要单独警惕的，是少数样本在局部层面明显过近，已经接近可识别的来源改写。

### 6.6 最终回答

如果把整个项目压缩成几句最直接的话，那么答案大概是这样的。

第一，AI 已经可以写出一部分基本成立的宋词，但这种能力远不稳定，尤其经不起长调和强主题的持续压力。第二，不同模型差异很大，在本项目所测范围内，`doubao-seed-2.0-pro` 是最稳的一档，`deepseek-3.2-thinking` 内容反应更强但风险也更高，`qwen3.6-plus` 则更容易在长调里失控。第三，prompt 确实能显著改变结果，但“更强”不等于“更好”：强约束会提升可控性，也会快速压缩内容空间。第四，AI 宋词确实存在明显的同质化倾向，而且这种倾向在主题任务和批量生成中会被迅速放大。第五，它与真实宋词之间的关系，大多数时候不是整首照抄，而是对传统词境、短句资源和成熟表达方式的高密度再调用；少数局部过近的样本，则需要被单独识别和复核。

因此，这个项目最后得到的并不是“AI 会不会写词”这样一个简单的结论，而是一条更具体的判断：AI 已经具备了有限而可操控的宋词生成能力，但这种能力高度依赖模型底座、prompt 设计和后续校验流程。它可以成为一个生成引擎，却还远远不是一个可以放心交付的“词人”。
