# 新版pre竖向重构稿

## 二、按任务 prompt 逐个观察生成结果

### 2.1 `a1`：自由生成 prompt

这一组任务是整个项目的起点。它不给模型规定词牌，也不给模型规定主题，只要求它自己写出一首“像样的宋词”。因此，`a1` 最重要的意义，不是看模型能不能完成某个指定任务，而是先看它在没有额外牵引时，会自然写成什么样。

#### 2.1.1 实际使用的 prompt 原文

~~~text
你现在是一位精通宋词的中国古典文学专家，也是一个数据生成引擎。
你需要自己选择一个词牌，创作一首符合格律的完整的宋词。请务必注意所作词的字数，格律（使用词林正韵、使用钦定词谱）的准确性。内容希望尽可能多样，不能千篇一律。希望能有一定创新性。请尽可能不要抄袭。

【输出要求】
你必须严格输出且仅输出合法的 JSON 格式数据，不要包含任何额外的解释、问候或 markdown 标记（如 ```json）。

【字段及约束说明】
1. "cipai" (字符串): 词牌名。
2. "title" (字符串): 词的自拟题目。
3. "content" (字符串): 词的正文。请严格遵循该词牌的字数和句式，句子之间用全角逗号“，”或句号“。”分隔，不要包含换行符。


现在，请开始生成。
~~~

#### 2.1.2 样本展示

`doubao-seed-2.0-pro`，`鹧鸪天`，`夏末过郊野残塘`，格律分 `100`

```text
岸柳牵风浅著黄，残荷擎雨暗留香。轻鸥点破波心碧，老蛩吟残陌上霜。
云影淡，客愁长，年光容易误疏狂。欲寻旧梦浑无迹，只剩寒塘送夕阳。
```

`deepseek-3.2-thinking`，`蝶恋花`，`数字时代感怀`，格律分 `100`

```text
网络连天云卷处，信息纷飞，似雪随风舞。数据海洋深几许，人机对话新情绪。
代码如诗情未露，算法精微，幻化千般路。古韵新声相映辅，时光流转心长驻。
```

#### 2.1.3 先看整体结果

| 项目 | `a1` 整体表现 |
|---|---|
| 样本总数 | `402` |
| 平均格律分 | `70.37` |
| 满分率 | `49.0%` |
| 高质量比例 | `59.7%` |
| `0` 分比例 | `19.9%` |
| 最常见词牌 | `鹧鸪天(97)`、`青玉案(70)`、`临江仙(32)` |
| 最常见意象 | `西风(67)`、`孤舟(55)`、`芦花(52)`、`明月(47)`、`星河(34)` |
| 最常见类型 | `离别羁旅(138)`、`闲适隐逸(59)`、`豪情壮志(41)` |

这张表先给出三个直接印象。第一，所谓“自由生成”并不等于真正发散，模型很快会回到少数自己熟悉的词牌。第二，意象并不杂乱无章，而是明显偏向风月、舟楫、秋江、远望这一组传统词语。第三，在不指定题目的情况下，模型最容易自然落向的题材，不是议论性、叙事性很强的作品，而是更常见的羁旅感伤一路。

换句话说，`a1` 已经暴露出一个很重要的事实：模型即使没有被要求“写成某种词”，也会主动把自己收束到少数默认路径中。

#### 2.1.4 三个主力模型的默认状态

| 模型 | 样本 | 平均格律分 | 默认词牌 | 意象倾向 | 类型倾向 |
|---|---:|---:|---|---|---|
| `doubao-seed-2.0-pro` | `50` | `94.32` | `鹧鸪天(26)`、`卜算子(18)`、`浣溪沙(5)` | `清露`、`岸柳`、`石径`、`晚风`、`野菊` | `闲适隐逸(14)`、`山水田园(11)`、`离别羁旅(8)` |
| `deepseek-3.2-thinking` | `50` | `69.52` | `鹧鸪天(15)`、`青玉案(11)`、`蝶恋花(5)` | `数据`、`代码`、`算法`、`星河`、`键盘` | `咏物寄托(13)`、`悼亡相思(9)`、`豪情壮志(7)` |
| `qwen3.6-plus` | `49` | `64.49` | `鹧鸪天(21)`、`青玉案(8)`、`临江仙(8)` | `孤舟`、`雁字`、`西风`、`芦花`、`星斗` | `离别羁旅(29)`、`闲适隐逸(8)`、`豪情壮志(4)` |

从格律表现看，三者一开始就拉开了差距。`doubao` 在自由状态下已经非常稳，平均分 `94.32`，说明它会主动选择自己能驾驭的词牌；`deepseek` 和 `qwen3.6-plus` 则明显没那么稳，尤其 `qwen3.6-plus` 在自由任务里也已经出现了较高的失误率。

从内容倾向看，三者的“默认宋词观”也不一样。`doubao` 更像在写传统景物词，常用的是清露、岸柳、石径、晚风这一类不太冒险的古典语汇；`deepseek` 最特别，它会把 `数据`、`代码`、`算法` 这类现代词直接带进词里，这种写法确实有辨识度，但也最容易让读者感到它并没有完全进入宋词语境；`qwen3.6-plus` 则最容易迅速滑回到羁旅感伤这条旧路上，孤舟、雁字、西风、芦花几乎很快就组成了它的默认场景。

#### 2.1.5 补充观察

补充模型里也能看到类似现象。`qwen3-max` 的平均格律分达到 `93.17`，看上去很强，但它的 `48` 首样本中有 `39` 首都写成了 `青玉案`，类型上也几乎压到 `离别羁旅` 一路，这说明它的高分部分建立在很强的默认收束上。相较之下，`glm-5` 的分布更散，但稳定性不足，说明“会换词牌”和“能写稳”并不是一回事。

### 2.2 `b1`：固定五词牌、无主题 prompt

`b1` 比 `a1` 只多了一层限制：研究者不再让模型自己挑词牌，而是固定在五个词牌中生成。但主题仍然不规定。也正因为如此，`b1` 很适合拿来观察“形式约束”本身的作用，看词牌一旦被指定，模型的格律、意象和题材会如何变化。

#### 2.2.1 实际使用的 prompt 原文

~~~text
你现在是一位精通宋词的中国古典文学专家，也是一个数据生成引擎。
你需要使用我提供的【词牌名】，创作一首符合格律的完整的宋词,使用词林正韵,使用钦定词谱。请务必注意所作词的字数，保证要输出完整的一首词，并注意格律的准确性。内容希望尽可能多样，不能千篇一律。希望能有一定创新性。请尽可能不要抄袭。


【输出要求】
你必须严格输出且仅输出合法的 JSON 格式数据，不要包含任何额外的解释、问候或 markdown 标记（如 ```json）。

【字段及约束说明】
1. "cipai" (字符串): 词牌名。
2. "title" (字符串): 词的自拟题目。
3. "content" (字符串): 词的正文。请严格遵循该词牌的字数和句式，句子之间用全角逗号“，”或句号“。”分隔，不要包含换行符。

【示例】
输出：
{
  "cipai": "如梦令",
  "title": "春晚",
  "content": "昨夜雨疏风骤，浓睡不消残酒。试问卷帘人，却道海棠依旧。知否，知否？应是绿肥红瘦。",
}

现在，请开始生成。输入：词牌名：《{cipai_name}》
~~~

实际运行时，最后一行中的 `{cipai_name}` 会被分别替换成 `清平乐`、`浪淘沙`、`菩萨蛮`、`沁园春`、`祝英台近`。也就是说，这一组任务关心的不是“宋词大体写得怎么样”，而是“同样不给主题时，不同词牌本身会把结果推向哪里”。

#### 2.2.2 样本展示

`doubao-seed-2.0-pro`，`浪淘沙`，`客中秋望`，格律分 `100`

```text
霜叶满汀湾，雁带秋寒。西风吹袂倚阑干。望极遥天烟水阔，何处家山。
客鬓已催残，旧约都闲。孤灯酒醒漏声阑。待得东君传信早，同醉花间。
```

#### 2.2.3 五个词牌的总体差异

| 词牌 | 样本 | 平均格律分 | 满分率 | 高质量比例 | `0` 分比例 |
|---|---:|---:|---:|---:|---:|
| `浪淘沙` | `125` | `84.10` | `74.4%` | `80.8%` | `12.0%` |
| `清平乐` | `136` | `84.00` | `71.3%` | `77.9%` | `11.8%` |
| `菩萨蛮` | `148` | `72.43` | `53.4%` | `58.8%` | `14.2%` |
| `祝英台近` | `135` | `62.81` | `39.3%` | `52.6%` | `27.4%` |
| `沁园春` | `142` | `56.90` | `36.6%` | `46.5%` | `31.7%` |

这张表已经把词牌差异说得很清楚。`清平乐` 和 `浪淘沙` 是明显的“稳区”，模型写起来最不容易崩；`菩萨蛮` 虽然还在可控范围内，但已经没那么稳；`祝英台近` 和 `沁园春` 则明显进入高压区，其中《沁园春》最难。也就是说，`b1` 的意义不在于让所有模型普遍变好，而在于第一次把“词牌难度”这件事直接摆到了台面上。

和 `a1` 相比，这一步最大的变化也不在于整体均分，而在于词牌一旦固定，模型更容易迅速收束到少数熟悉写法中。

#### 2.2.4 不同词牌把内容拉向哪里

| 词牌 | 代表意象 | 主要类型 |
|---|---|---|
| `清平乐` | `明月`、`西风`、`斜阳`、`清风`、`孤舟` | `闲适隐逸(47)`、`离别羁旅(32)`、`山水田园(20)` |
| `浪淘沙` | `孤舟`、`烟水`、`荻花`、`雁字`、`明月` | `离别羁旅(79)`、`闲适隐逸(16)`、`怀古咏史(13)` |
| `菩萨蛮` | `明月`、`孤舟`、`芦花`、`烟波`、`西风` | `离别羁旅(90)`、`悼亡相思(15)`、`春怨秋悲(13)` |
| `沁园春` | `明月`、`雁阵`、`斜阳`、`西风`、`芦花` | `怀古咏史(51)`、`豪情壮志(35)`、`闲适隐逸(28)` |
| `祝英台近` | `暮云`、`西风`、`明月`、`雁字`、`阑干` | `离别羁旅(80)`、`悼亡相思(28)`、`春怨秋悲(14)` |

从内容上看，五个词牌并不是简单地“换了个外壳”。`清平乐` 更容易把模型拉向清景、闲情、小令式的抒写，所以它的主类型是 `闲适隐逸`；`浪淘沙` 很快就把内容推向江天远望、舟中怀人的羁旅场景；`菩萨蛮` 和 `祝英台近` 也偏感伤，但前者更容易滑向秋江晚景，后者则更容易形成栏畔回望、欲说还休的凝结情绪。

最特别的是《沁园春》。它的格律最难稳住，但内容并没有像其他几个词牌那样大量压到羁旅感伤一路，反而更容易转向 `怀古咏史` 和 `豪情壮志`。这说明《沁园春》对模型造成的影响，不只是“更难”，也是“更容易把内容往大开大合、议论放言的方向带”。

### 2.3 `b2`：五词牌 + 改革主题 prompt

到了 `b2`，词牌仍然固定在前面的五个词牌中，但 prompt 又额外加入了“宗门改革”的完整情境。也就是说，模型此时不只是要把词写成某一词牌，还要把词写成一个处在制度压迫与改革犹疑中的人物之作。

#### 2.3.1 实际使用的 prompt 原文

下面列出一条实际输入的完整 prompt。`b2` 在运行时会在五个词牌之间轮换；这里以《清平乐》这一条为例，其他几条仅词牌名不同，主题内容完全相同。

~~~text
你现在是一位精通宋词的中国古典文学专家，也是一个数据生成引擎。
你需要使用我提供的【词牌名】和【主题和背景】，创作一首符合格律的完整的宋词，使用词林正韵，最好使用钦定词谱。请务必注意所作词的完整性和格律的准确性。你需要在主题和背景的框架下充分发挥创造力，体现文学的美，努力学习使用词人常用的抒情手法。希望能有一定创新性。请尽可能不要抄袭。

【输出要求】
你必须严格输出且仅输出合法的 JSON 格式数据，不要包含任何额外的解释、问候或 markdown 标记（如 ```json）。

【字段及约束说明】
1. "cipai" (字符串): 词牌名。
2. "title" (字符串): 词的自拟题目。
3. "content" (字符串): 词的正文。请严格遵循该词牌的字数和句式，句子之间用全角逗号“，”或句号“。”分隔，不要包含换行符。

【示例】
输出：
{
  "cipai": "如梦令",
  "title": "春晚",
  "content": "昨夜雨疏风骤，浓睡不消残酒。试问卷帘人，却道海棠依旧。知否，知否？应是绿肥红瘦。",
}

现在，请开始生成。输入：词牌名：《清平乐》，主题和背景：假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。
~~~

与 `b1` 相比，这里新增的不是一个抽象的“改革”标签，而是一整段带有人物身份、制度冲突和心理挣扎的写作情境。它要求模型把形式、场景和立场同时组织起来，因此比 `b1` 明显更难。

#### 2.3.2 样本展示

`doubao-seed-2.0-pro`，`清平乐`，`玄峰观规有感`，格律分 `100`

```text
霜钟催鼓，束尽青衿步。旧例森严传自古，可护玄门寒暑。
诸公各执陈章，谁怜少俊摧藏？我欲裁冰剪雨，又恐寒透云廊。
```

#### 2.3.3 先看整体结果

| 项目 | `b2` 整体表现 |
|---|---|
| 样本总数 | `628` |
| 平均格律分 | `63.03` |
| 满分率 | `40.1%` |
| 高质量比例 | `50.6%` |
| `0` 分比例 | `24.0%` |
| 与 `b1` 简比 | 均分 `-8.71`；高质量比例 `-12.2` 个百分点；`0` 分比例 `+4.5` 个百分点 |
| 最常见意象 | `樊笼(71)`、`劫火(54)`、`灵根(51)`、`剑气(48)`、`铁律(48)` |
| 最常见类型 | `谈禅说理(258)`、`咏物寄托(249)`、`豪情壮志(121)` |

和 `b1` 相比，`b2` 最直观的变化，是模型不再自然滑向羁旅感伤一路，而是被这组改革叙事强行拉进新的词汇场。`樊笼`、`劫火`、`灵根`、`剑气`、`铁律` 这些词已经说明，主题压力一旦加入，意象池就不再由词牌单独决定，而开始被情境本身改写。

同时，这种改写并不是免费的。`b2` 的平均格律分比 `b1` 低了 `8.71` 分，`0` 分比例也明显上升。这说明主题进入之后，模型确实更容易顾此失彼。

#### 2.3.4 三个主力模型的响应方式

| 模型 | 样本 | 平均格律分 | 意象倾向 | 类型倾向 |
|---|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `200` | `81.08` | `樊笼`、`劫火`、`松风`、`雏鹰`、`云` | `谈禅说理(126)`、`咏物寄托(66)`、`豪情壮志(8)` |
| `deepseek-3.2-thinking` | `200` | `51.92` | `春风`、`晨钟`、`暮鼓`、`宗门`、`剑气` | `咏物寄托(78)`、`谈禅说理(77)`、`豪情壮志(45)` |
| `qwen3.6-plus` | `161` | `59.43` | `铁律`、`青衿`、`星斗`、`剑阁`、`灵根` | `咏物寄托(67)`、`豪情壮志(51)`、`谈禅说理(43)` |

三款主力模型对这条 prompt 的响应方向并不一样。`doubao` 仍然保持了明显的格律优势，平均分还有 `81.08`，说明它在“加了主题之后”仍能把形式守住不少；但它的内容明显转向说理色彩最强的一路，`谈禅说理` 占到 `126` 首，几乎成了它理解“改革主题”的主要方式。`deepseek` 的格律压力最大，均分掉到 `51.92`，但在内容上反而最摇摆，`咏物寄托` 和 `谈禅说理` 几乎并列。`qwen3.6-plus` 则更容易把这类主题写成带有立志、冲决意味的作品，所以 `豪情壮志` 的比重明显高于另外两家。

也就是说，同样面对“宗门改革”这一情境，不同模型理解它的方式并不一致。有的把它写成说理，有的把它写成象征寄托，有的更倾向把它写成带冲突感的抒怀词。

#### 2.3.5 补充观察

补充样本里，`glm-5` 的平均格律分只有 `47.87`，比三家主力模型更低，但它的内容方向也很有代表性：`樊笼`、`剑气`、`清规` 这些词出现频率很高，说明在这类主题下，它虽然形式失稳，却也会迅速被拉进宗门、规训、抗争这一套设定化语汇之中。

### 2.4 `b3`：五词牌 + 玉兰主题 prompt

`b3` 和 `b2` 结构相同，仍然是五词牌轮换，但主题从“宗门改革”换成了“大学生在白玉兰前重新找到生命价值”。因此，这一节最值得看的，不只是它和 `b1` 的差异，也包括它和 `b2` 的差异：同样是在五词牌里加主题，两类主题会把内容推向不同方向。

#### 2.4.1 实际使用的 prompt 原文

从内容看，三家其实都没有偏离主题。`晨钟`、`暮鼓`、`宗门`、`弟子`、`铁律`、`灵根` 这些词都很清楚地落在改革情境里。问题恰恰在于：它们在“知道要写什么”的同时，没有同样稳定地“知道怎么把它写成《沁园春》”。这也正是 `c2` 与 `c1` 的根本区别所在。

## 三、从格律、意象、类型三个方向总结整个任务链

到了这一章，重点不再是逐条展示 prompt 本身，而是把第二章分散呈现的数据重新收束成三条更清楚的线索：形式如何变化，意象如何改写，类型如何收束。这里不再重复第二章已经展示过的模型逐项结果，而只回答整个任务链最重要的变化规律。

### 3.1 格律方向总结

先把整条任务链的格律变化压缩成一张表：

| 任务 | 平均格律分 | 高质量比例 | `0` 分比例 | 字数不符比例 | 应平实仄比例 | 应仄实平比例 |
|---|---:|---:|---:|---:|---:|---:|
| `a1` | `70.37` | `59.7%` | `19.9%` | `11.4%` | `35.1%` | `28.6%` |
| `b1` | `71.74` | `62.8%` | `19.5%` | `14.7%` | `24.3%` | `26.8%` |
| `b2` | `63.03` | `50.6%` | `24.0%` | `18.8%` | `36.9%` | `31.7%` |
| `b3` | `65.55` | `54.7%` | `25.2%` | `19.5%` | `31.3%` | `27.2%` |
| `c1` | `61.79` | `53.5%` | `31.4%` | `10.2%` | `44.8%` | `35.9%` |
| `c2` | `44.24` | `34.1%` | `45.1%` | `14.0%` | `60.2%` | `49.2%` |

这张表可以帮助我们把几个关键变化分开来看。

第一，从 `a1` 到 `b1`，格律整体并没有发生剧烈变化，但确实略有改善。平均分从 `70.37` 升到 `71.74`，高质量比例也从 `59.7%` 升到 `62.8%`。这说明单纯把词牌固定下来，确实能给模型一个基本的形式支架。换句话说，词牌约束本身不是负担，它在不少时候反而能减少模型在“写哪种词”上的摇摆。

第二，真正明显的下滑出现在主题进入之后。`b2` 和 `b3` 的平均分都低于 `b1`，其中改革主题的 `b2` 跌得更明显，均分降到 `63.03`。这说明主题不是一个附加说明，而是会真实占用模型的组织能力。模型在同时处理人物身份、处境冲突和抒情方向时，更容易在形式控制上失手。

第三，长调测试把这种压力进一步放大。`c1` 和 `c2` 都固定到《沁园春》，但两者表现并不一样。`c1` 虽然平均分只有 `61.79`，但至少仍然维持在一个可读区间；`c2` 则直接跌到 `44.24`，`0` 分比例高达 `45.1%`。这说明《沁园春》真正困难的地方，不只在于它是长调，更在于长调需要持续、完整的结构控制。只提醒“注意词林正韵、注意钦定词谱”并不能替代显式模板。

如果把错误类型拆开看，规律会更清楚。`c1` 的一个值得注意的特点是：它的字数不符比例只有 `10.2%`，反而低于 `b2` 和 `b3`。这说明显式模板确实能帮助模型守住整体篇幅和句式外框。但与此同时，`c1` 的平仄错误却急剧升高，`应平实仄` 达到 `44.8%`，`应仄实平` 达到 `35.9%`。也就是说，模板可以帮模型“写够”，却不能保证它“逐句写准”。

`c2` 则更极端。它的字数不符并不是最严重的，但平仄链条几乎全面失守：`应平实仄` 达到 `60.2%`，`应仄实平` 达到 `49.2%`。这说明到了“改革主题 + 长调 + 无显式模板”的组合里，模型最先崩掉的不是押韵或个别句子，而是整条平仄链的持续控制能力。

因此，从格律这一条线看，整条任务链给出的结论并不复杂：词牌约束本身有助于稳定形式，主题压力会挤占形式控制，长调会把这种压力成倍放大，而显式模板只能部分缓解这种放大，不能彻底消除它。

### 3.2 意象方向总结

和格律相比，意象这条线的变化更快，也更明显。模型并不是先大幅改变题材，再慢慢改变意象；相反，很多时候意象池是最先发生重组的。

先看数据库之间的重合度：

| 对比组 | 共享意象 Jaccard | Top20 重合数 | 说明 |
|---|---:|---:|---|
| `a1` vs `b1` | `0.256` | `15` | 词牌固定后，默认意象仍大量延续 |
| `b1` vs `b2` | `0.106` | `2` | 改革主题大幅改写原有意象池 |
| `b1` vs `b3` | `0.127` | `1` | 玉兰主题几乎重建意象中心 |
| `b3` vs `c1` | `0.275` | `16` | 固定《沁园春》后，玉兰主题意象仍高度延续 |
| `b2` vs `c2` | `0.219` | `14` | 改革主题进入《沁园春》后，核心意象大体保留 |

这张表说明，意象变化并不是平均发生的。`a1` 和 `b1` 的 top20 意象有 `15` 个重合，说明从“自由生成”到“固定五词牌”，模型依然基本活在同一套常见词语里，无非是把这些词放进更稳定的词牌框架中。也就是说，词牌能改变组织方式，但不一定立刻改写底层意象池。

真正的改写发生在主题进入之后。无论是 `b2` 还是 `b3`，与 `b1` 的 top20 重合都几乎消失：前者只剩 `2` 个，后者只剩 `1` 个。这一点非常关键，因为它说明主题不是在原有意象池上做轻微修饰，而是在相当大程度上重建了一套新的高频词汇。

各数据库的高频意象也能把这种改写过程说得更具体：

| 任务 | 最常见意象前 5 |
|---|---|
| `a1` | `西风`、`孤舟`、`芦花`、`明月`、`星河` |
| `b1` | `明月`、`孤舟`、`西风`、`暮云`、`雁字` |
| `b2` | `樊笼`、`劫火`、`灵根`、`剑气`、`铁律` |
| `b3` | `玉兰`、`寒窗`、`素萼`、`玉树`、`暗香` |
| `c1` | `玉兰`、`琼枝`、`寒窗`、`琼英`、`玉树` |
| `c2` | `青衿`、`灵根`、`樊笼`、`丹炉`、`晨钟` |

从这张表可以看出，`a1` 和 `b1` 的默认意象池主要还是风月、舟楫、秋江、远望这一套传统词汇。到了 `b2`，这套默认池立刻被宗门改革叙事改写成另一组词：`樊笼`、`劫火`、`灵根`、`剑气`、`铁律`，都明显带着制度压抑、修行体系和冲突感。到了 `b3`，又换成以玉兰为中心的另一组词：`玉兰`、`素萼`、`玉树`、`暗香`、`寒窗`。

这说明词牌约束主要影响的是“怎么写”，而主题约束更直接影响“写什么”。这也是为什么第二章里一旦主题进入，读者会立刻感到作品面目发生了变化。

再往下看，`b3` 与 `c1` 的 top20 意象重合达到 `16` 个，而 `b2` 与 `c2` 也达到 `14` 个。这说明进入《沁园春》以后，主题本身并没有被洗掉。长调改变的更多是形式难度，而不是意象方向。换言之，到了长调任务里，模型通常还是知道自己应该围绕什么写，只是它未必有能力把这些内容稳定地安放进长调结构中。

这一条线还揭示出一个更细的差别：玉兰主题比改革主题更容易形成高度模板化的物象中心。`b3` 中 `玉兰` 一项就覆盖了 `36.4%` 的作品，到了 `c1`，`玉兰`、`琼枝`、`琼英`、`玉树`、`素萼` 这一串词几乎构成了一套稳定的替换链。改革主题虽然也会收束，但它的词汇中心更多分散在 `樊笼`、`灵根`、`青衿`、`晨钟`、`暮鼓`、`宗门` 等多个节点上，没有玉兰主题那样单核。

因此，从意象这条线看，最重要的结论有两点。第一，默认意象池确实存在，而且在无主题任务中非常稳定；第二，主题一旦进入，意象会被迅速改写，其中玉兰主题的收束更强、更像围绕中心物象组织，改革主题则更像围绕情境词群展开。

### 3.3 类型方向总结

如果说意象是“词里写了什么景和物”，那么类型分布更接近于“词最终被写成了什么作品”。这一条线的变化，比意象更适合看“收束”本身。

先看整条任务链的数据库级分布：

| 任务 | 第一类型 | 占比 | 第二类型 | 占比 | 第三类型 | 占比 |
|---|---|---:|---|---:|---|---:|
| `a1` | `离别羁旅` | `34.3%` | `闲适隐逸` | `14.7%` | `豪情壮志` | `10.2%` |
| `b1` | `离别羁旅` | `42.7%` | `闲适隐逸` | `15.7%` | `怀古咏史` | `9.8%` |
| `b2` | `谈禅说理` | `41.1%` | `咏物寄托` | `39.6%` | `豪情壮志` | `19.3%` |
| `b3` | `咏物寄托` | `92.0%` | `豪情壮志` | `5.2%` | `谈禅说理` | `1.9%` |
| `c1` | `咏物寄托` | `87.7%` | `豪情壮志` | `10.9%` | `离别羁旅` | `0.7%` |
| `c2` | `谈禅说理` | `57.2%` | `豪情壮志` | `32.6%` | `咏物寄托` | `9.8%` |

这张表把整个项目最清楚的一条线索直接摆了出来。

在无主题阶段，模型默认最容易把词写成 `离别羁旅`。`a1` 是 `34.3%`，到了 `b1` 还升到 `42.7%`。这说明只要不规定主题，模型对“宋词”最自然的理解，仍然是远行、怀人、秋景、感伤这一整套传统抒情模式。词牌固定以后，这一倾向没有减弱，反而更强。

改革主题进来以后，类型中心明显转移。`b2` 里，`离别羁旅` 已经不再是主轴，前两位变成 `谈禅说理` 和 `咏物寄托`。这说明这类题目很难再被写成纯粹的羁旅感伤词，而更容易被模型理解成“借物说理”“借处境立论”“以冲突写志”的作品。

玉兰主题的变化更激烈。`b3` 中 `咏物寄托` 一项就占到 `92.0%`，到了 `c1` 仍有 `87.7%`。这几乎已经不是“偏向”某种类型，而是被一种中心写法高度锁定。这也是为什么第二章里玉兰主题看起来特别整齐：它不是偶然相似，而是类型上已经几乎被压成同一路数。

`c2` 则提供了一个重要反例。它虽然也是固定《沁园春》，但由于主题换回改革叙事，类型分布立刻从玉兰线的 `咏物寄托` 转回 `谈禅说理 + 豪情壮志`。这说明决定类型中心的主要不是长调本身，而是主题本身。长调放大的更多是形式压力，而不是类型方向。

模型之间的差异，也可以更集中地看出来：

| 任务 | `doubao-seed-2.0-pro` | `deepseek-3.2-thinking` | `qwen3.6-plus` |
|---|---|---|---|
| `a1` | `闲适隐逸` 为首，`28.0%` | `咏物寄托` 为首，`26.0%` | `离别羁旅` 为首，`59.2%` |
| `b2` | `谈禅说理` 为首，`63.0%` | `咏物寄托` `39.0%`，`谈禅说理` `38.5%`，最摇摆 | `咏物寄托` 为首，`41.6%`，`豪情壮志` 次之 |
| `b3` | `咏物寄托` 为首，`91.5%` | `咏物寄托` 为首，`93.0%` | `咏物寄托` 为首，`91.5%` |
| `c1` | `咏物寄托` 为首，`84.0%` | `咏物寄托` 为首，`96.5%` | `咏物寄托` 为首，`81.2%` |
| `c2` | `谈禅说理` 为首，`80.0%` | `谈禅说理` `49.5%`，`豪情壮志` `40.4%` | `豪情壮志` 为首，`49.2%`，`谈禅说理` 次之 |

这张表揭示了三点。

第一，在没有主题时，`qwen3.6-plus` 最容易模板化地滑向 `离别羁旅`，这一点在 `a1` 就已经非常明显。它不是到了后面才开始收束，而是起点就比另外两家更窄。

第二，在改革主题下，三家模型并没有被锁到同一种类型里。`doubao` 最容易把它写成说理之作，`qwen3.6-plus` 更容易保留抒怀和立志色彩，`deepseek` 则常常在说理与寄托之间来回摆动。这说明改革主题虽然能改变默认类型，但它并不像玉兰主题那样会把所有模型都压进同一条窄路。

第三，玉兰主题才是整条任务链里收束最强的节点。到了 `b3` 和 `c1`，三家主力模型都被压到 `咏物寄托` 一路，比例普遍超过 `80%`，`deepseek` 甚至达到 `96.5%`。这说明玉兰主题几乎天然地带着强模板倾向，它会快速缩减模型的可选写法空间。

因此，从类型这条线看，整条任务链可以概括成一句话：无主题时默认落向羁旅感伤，改革主题把作品压向说理与立志，玉兰主题则几乎把作品锁定为咏物寄托。这其中，主题的力量明显大于词牌，而模型差异主要体现在“被主题压住之后，还能保留多少自己的默认偏向”。

### 3.4 本部分总结

把格律、意象、类型三条线放在一起看，第二章背后的总趋势就比较清楚了。

第一，prompt 改变生成，往往不是平均地改变所有层面，而是有先后顺序。词牌最先影响的是形式框架和局部内容惯性；主题最先改写的是意象池和类型中心；长调则把形式压力骤然放大。也就是说，模型不是同时在所有维度一起变化，而是在不同 prompt 下被分别推向不同的压力点。

第二，整条任务链里变化最大的一步，不是从 `a1` 到 `b1`，而是从无主题进入主题阶段，尤其是 `b1 -> b2` 和 `b1 -> b3`。前者几乎重建了意象池，后者几乎锁定了类型中心。到了 `c1` 和 `c2`，变化更多体现在形式崩塌程度，而不再是内容方向本身。

第三，最值得后文继续关注的风险，不是“模型会不会写成某一类词”，而是两类更深的问题。一类是形式层的持续失控，尤其是长调中的平仄链崩塌；另一类是内容层的过快收束，尤其是玉兰主题下几乎整齐划一的咏物寄托写法。前者会影响作品能否成立，后者则关系到批量生成是否迅速滑向模板化。
