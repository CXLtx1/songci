## 一、项目到底在做什么

### 1.1 核心问题

这个项目研究的，不是“AI 能不能写出像古诗词的句子”，而是更具体的五个问题：

1. AI 能不能写出在格律和结构上基本成立的宋词。
2. 在不同任务条件下，AI 会把词写成什么样。
3. AI 会稳定偏爱哪些意象、哪些题材、哪些写法。
4. AI 在批量生成时会不会越来越写成固定模板。
5. AI 生成的宋词与真实宋词之间，到底是风格接近、传统化用，还是出现了更值得警惕的局部借用。

换句话说，这个项目不是只看“像不像宋词”，而是同时看形式能力、内容组织、内部同质化和与真实作品的关系。

### 1.2 整个项目的流程

整个项目可以概括为六步：

1. 在 `generate/prompts.py` 中定义 prompt 模板。
2. 在 `generate/json/*.json` 中设定模型、数据库、词牌池和主题文本。
3. 批量调用模型生成 JSON 格式宋词，并写入 `database/*.db`。
4. 用 `engine/songci_engine.py` 对字数、平仄、押韵进行自动校验。
5. 为每首词抽取意象与主题类别，形成描述性统计。
6. 再进一步做两类深分析：一类是 AI 与真实宋词的来源相似性，一类是 AI 自身的内部同质化。


### 1.3 六类任务到底是什么

本项目设计了六组任务，它们不是随意分组，而是一条逐步加约束的链：

| 数据库 | 中文解释 | 任务性质 | Prompt 类型 |
|---|---|---|---|
| `a1` | 自由生成 | 不限词牌，不限主题 | `type_c` |
| `b1` | 五词牌、无主题 | 只加词牌约束 | `type_b` |
| `b2` | 五词牌、改革主题 | 词牌约束 + 主题约束 | `type_d` |
| `b3` | 五词牌、玉兰主题 | 词牌约束 + 主题约束 | `type_d` |
| `c1` | 固定 `沁园春`、玉兰主题、显式格律模板 | 强结构任务 | `type_a` |
| `c2` | 固定 `沁园春`、改革主题、简略格律要求 | 强主题 + 简略格律提示 | `type_e` |

这里有一个必须明确的地方：`c2` 不是“比 `c1` 更硬的格律模板任务”。`c1` 才是显式给出 `沁园春` 格律模板的一组；`c2` 是固定 `沁园春` 与改革主题，但只用较简短的格律与用韵提醒来约束模型。

### 1.4 Prompt 到底让 AI 做什么

如果只说 `type_a/type_b/type_c/type_d/type_e`，读者很容易迷失。因此这里先用人话解释：

- `type_c`：模型自己选词牌、自己决定主题，自由写一首词。
- `type_b`：词牌限定在给定范围里，但不规定主题。
- `type_d`：同时规定词牌和主题背景。
- `type_a`：固定 `沁园春`，并直接给出较完整的格律模板。
- `type_e`：固定 `沁园春`，强调词林正韵与钦定词谱，但不再展示完整格律骨架。

两个主题文本也必须还原成具体情境，而不是抽象标签：

- 改革主题：一个修仙宗门中的年轻长老，看见旧规压抑弟子，想改革却又担心危机中失去秩序与战力，于是借词抒怀。
- 玉兰主题：一个大学生在学业与面试中受挫，看见盛开的白玉兰后重新感到生命的实在与价值，于是写词。

### 1.5 样本规模与模型覆盖

六个 AI 数据库一共收录 `3122` 首词。逐库计数如下：

| `b1` | 686 |
| `b2` | 628 |

从横向比较角度，真正适合作为主比较对象的，是三家覆盖最完整、样本量也最大的模型：

## 二、AI 写的宋词对不对：格律与结构能力

### 2.1 这里的“对不对”到底指什么

这一部分说的“对不对”，主要是形式层，而不是文学高下。项目实际检查的是四件事：

1. 字数是否符合词牌要求。
2. 平仄是否基本成立。
3. 韵脚是否押对。
4. 长调里结构能不能一路维持下去。

也就是说，这里讨论的是“这首词先能不能站住”，而不是“它是不是已经写得像苏辛周姜”。

### 2.2 格律分是怎么来的

`engine/songci_engine.py` 的逻辑很清楚：先从 `100` 分出发，每出现一种格律问题就按权重扣分，最后按

```text
score = max(0, 100 - total_penalty × 4)
```

计算格律分。

程序里的扣分梯度大体是：

- 平仄错误：按较高权重扣分。
- 韵脚声调错误：比普通平仄更重。
- 出韵、挤韵：惩罚最高。
- 未收录字：只做警告，不扣分。

此外，字数不符会直接让该变体校验失败，所以很多 0 分并不是“平仄有一点瑕疵”，而是结构本身已经没对上。

### 2.3 先看 prompt 任务线：一加主题，格律分就会掉

如果不先讲 `a1/b1/b2/b3/c1/c2`，而是按任务条件来看，会更清楚：

| 任务说明 | 对应代码 | 平均格律分 | 100 分比例 | 高质量比例 | 0 分比例 |
|---|---|---:|---:|---:|---:|
| 完全自由生成 | `a1` / `type_c` | 70.37 | 49.0% | 59.7% | 19.9% |
| 固定五个词牌，不给主题 | `b1` / `type_b` | 71.74 | 54.5% | 62.8% | 19.5% |
| 固定五个词牌，给改革主题 | `b2` / `type_d` | 63.03 | 40.1% | 50.6% | 24.0% |
| 固定五个词牌，给玉兰主题 | `b3` / `type_d` | 65.55 | 44.3% | 54.7% | 25.2% |
| 固定 `沁园春`，给玉兰主题，并附显式词谱模板 | `c1` / `type_a` | 61.79 | 41.6% | 53.5% | 31.4% |
| 固定 `沁园春`，给改革主题，只做简略格律提醒 | `c2` / `type_e` | 44.24 | 22.7% | 34.1% | 45.1% |

这里有两个必须突出说的结论：

第一，主题一加上去，格律分就明显下降。  
从“固定五个词牌，不给主题”的 `71.74` 分，降到“固定五个词牌 + 改革主题”的 `63.03` 分，以及“固定五个词牌 + 玉兰主题”的 `65.55` 分，下降是非常明显的。说明模型一旦要同时兼顾具体情境、人物身份和叙事背景，形式控制就会被挤压。

第二，给了格律提示也并不意味着就能写好。  
`c1` 虽然已经直接给出 `沁园春` 的显式词谱模板，但平均格律分也只有 `61.79`，高质量比例只有 `53.5%`，0 分比例仍有 `31.4%`。  
也就是说，模板有帮助，但绝不是“给了模板就稳了”。这正是报告里必须明确说出来的地方。

### 2.4 同样是 `沁园春`，不给模板、给简略提示、给显式模板，到底差多少

为了把“词谱模板有没有用”讲清楚，不能只看整库平均分，还必须单拎 `沁园春` 看。

下表把所有 `沁园春` 任务放在同一条线上：

| `沁园春` 任务说明 | 对应代码 | 平均格律分 | 高质量比例 | 100 分比例 | 0 分比例 |
|---|---|---:|---:|---:|---:|
| 五词牌内任选 `沁园春`，无主题、无模板 | `b1` 中 `沁园春` | 56.90 | 46.5% | 36.6% | 31.7% |
| 五词牌内任选 `沁园春`，改革主题、无模板 | `b2` 中 `沁园春` | 56.93 | 43.0% | 29.9% | 28.0% |
| 五词牌内任选 `沁园春`，玉兰主题、无模板 | `b3` 中 `沁园春` | 56.86 | 45.7% | 32.8% | 33.6% |
| 固定 `沁园春`，玉兰主题，显式词谱模板 | `c1` | 61.65 | 53.4% | 41.3% | 31.5% |
| 固定 `沁园春`，改革主题，简略格律提醒 | `c2` | 44.24 | 34.1% | 22.7% | 45.1% |

这一组对比非常重要：

1. 显式模板确实有帮助，但帮助有限。  
同样写玉兰主题的 `沁园春`，从“五词牌内选 `沁园春`、不给模板”的 `56.86` 分，提升到“固定 `沁园春`、附显式词谱模板”的 `61.65` 分，只是有限提升，并没有接近“稳妥正确”。

2. 简略提醒不够。  
同样是改革主题，`b2` 里写成 `沁园春` 的平均分还有 `56.93`，到了“固定 `沁园春` + 简略格律提醒”的 `c2`，反而掉到 `44.24`。这说明只说“注意词林正韵、注意钦定词谱”并不能替代显式模板。

3. 模板不是万能药。  
`c1` 的 0 分比例仍有 `31.5%`，而且不同模型吃到模板后的收益完全不一样。

### 2.5 模型差异：模板和主题对不同模型的作用不一样

三家主模型的整体格律表现依然差距很大：

| 任务说明 | `doubao-seed-2.0-pro` | `deepseek-3.2-thinking` | `qwen3.6-plus` |
|---|---:|---:|---:|
| 完全自由生成 | 94.32 | 69.52 | 64.49 |
| 五词牌、无主题 | 92.12 | 68.80 | 61.98 |
| 五词牌、改革主题 | 81.08 | 51.92 | 59.43 |
| 五词牌、玉兰主题 | 87.04 | 55.02 | 53.59 |
| `沁园春` + 玉兰 + 显式模板 | 74.66 | 63.48 | 41.72 |
| `沁园春` + 改革 + 简略提醒 | 79.76 | 18.59 | 28.68 |

如果只盯总表，容易错过一个更关键的事实：**模板的作用也强烈依赖模型。**

就 `沁园春` 而言：

- `deepseek` 在玉兰主题下，从无模板的 `33.94` 分提升到显式模板下的 `63.48` 分，说明它非常依赖外部骨架。
- `qwen3.6-plus` 在玉兰主题下，从无模板的 `45.89` 分到显式模板的 `41.72` 分，几乎没有得到帮助，说明“看见模板”不等于“能把模板执行好”。
- `doubao` 在五词牌玉兰主题下写 `沁园春` 已经有 `83.47` 分，到 `c1` 反而变成 `74.66` 分，说明它原本就能维持较强结构，模板不一定继续加分。

这正好说明：模型变量在格律问题里非常关键。  
同样的模板、同样的词牌、同样的主题，不同模型的收益完全不同。

### 2.6 错误主要错在哪里

从错误类型看，最主要的不是押韵，而是平仄和长调结构持续性。

数据库级统计里，最值得注意的是：

- `b1` 中“应平实仄”覆盖 `24.3%` 作品，“应仄实平”覆盖 `26.8%`。
- 改革主题 prompt 下，这两个比例升到 `36.9%` 与 `31.7%`。
- 显式模板的 `c1` 中，又进一步升到 `44.8%` 与 `35.9%`。
- 简略格律提醒的 `c2` 最严重，分别达到 `60.2%` 与 `49.2%`。

这说明在长调与强主题条件下，模型最先失守的往往不是“押不押韵”，而是“整首的平仄和句式能不能一路撑住”。

### 2.7 词牌本身也有难度梯度

如果把全部样本合并看，词牌之间的难度差异很明显：

| 词牌 | 平均格律分 | 高质量比例 | 0 分比例 |
|---|---:|---:|---:|
| `清平乐` | 81.33 | 74.0% | 13.1% |
| `浪淘沙` | 78.18 | 73.4% | 16.5% |
| `菩萨蛮` | 67.07 | 49.5% | 16.5% |
| `沁园春` | 56.28 | 46.5% | 34.5% |
| `祝英台近` | 50.29 | 38.8% | 37.7% |

这说明“AI 会不会写宋词”不能抽象成一个统一问题。它很可能能把 `清平乐`、`浪淘沙` 写得有模有样，却在 `祝英台近` 和 `沁园春` 上明显掉队。

### 2.8 本部分结论

这一部分最需要强调的结论有四个：

1. 主题一旦限定，格律正确率会大幅下降，这是项目里非常稳定的现象。
2. 显式词谱模板有帮助，但远远谈不上“给了就完美”。
3. 对 `沁园春` 这类长调来说，简略格律提醒明显不够，`c2` 是最直接的反例。
4. 格律问题上，模型差异非常关键：`doubao` 形式最稳，`deepseek` 最依赖外部骨架，`qwen` 则在长调上最容易掉链子。

## 三、AI 喜欢用什么意象

### 3.1 默认意象：不规定主题时，AI 最爱写什么

先看两类弱约束任务：

- 完全自由生成：模型自己选词牌、自己定题。
- 固定五个词牌、不规定主题：限定形式，但不规定内容。

在这两类 prompt 下，AI 的高频意象非常稳定。

完全自由生成时，最高频的是：

| 意象 | 覆盖比例 |
|---|---:|
| 西风 | 16.7% |
| 孤舟 | 13.7% |
| 芦花 | 12.9% |
| 明月 | 11.7% |
| 星河 | 8.5% |
| 烟波 | 8.2% |

固定五个词牌、不规定主题时，则变成：

| 意象 | 覆盖比例 |
|---|---:|
| 明月 | 15.7% |
| 孤舟 | 12.7% |
| 西风 | 12.0% |
| 暮云 | 10.6% |
| 雁字 | 10.3% |
| 斜阳 | 8.9% |

这说明所谓“自由生成”并不真的自由。不给主题时，模型会很快滑回一套高度传统、也高度成熟的宋词意象池：西风、孤舟、雁字、斜阳、暮云、烟波、芦花，几乎构成默认起手式。

### 3.2 不同词牌会把模型导向不同的意象

固定五个词牌后，词牌本身也会改变意象分布：

- `菩萨蛮`：明月、孤舟、芦花、西风、秋江，整体更清婉。
- `清平乐`：明月、西风、斜阳、清风、流水，偏短景与轻灵。
- `浪淘沙`：孤舟、烟水、荻花、雁字、暮色，更容易滑向羁旅江景。
- `祝英台近`：暮云、西风、明月、阑干、孤棹、晚烟，局部景物更密。
- `沁园春`：明月、雁阵、斜阳、芦花、落日，更偏大景和高空视角。

也就是说，词牌并不只是格律框架，它还会实际改变模型选择哪些景物、使用怎样的情绪姿态。

### 3.3 不同模型最喜欢什么意象

三家主模型的默认意象偏好并不一样。

#### `doubao`

`doubao` 更偏园亭山水和清疏小景。  
在完全自由生成时，常见词有 `清露`、`岸柳`、`石径`、`晚风`、`野菊`、`闲云`；固定五词牌后，又变成 `西风`、`疏钟`、`汀洲`、`篱菊`、`孤篷`、`归鸿`。

它的总体气质是“比较古典、比较稳、现代残留少”。

#### `deepseek`

`deepseek` 的特征最明显。  
在完全自由生成时，它会频繁出现 `数据`、`代码`、`算法`、`键盘`、`网络` 这类现代语词；固定五词牌后，虽然整体被拉回 `明月`、`暮云`、`青山`、`斜阳` 等传统词汇，但 `数据`、`代码` 仍然能进入高频列。

这说明它能模仿古典抒情框架，但更容易带出当代概念残留。

#### `qwen3.6-plus`

`qwen3.6-plus` 的默认意象最稳定地落在离愁羁旅轨道上。  
无论是完全自由生成还是固定五词牌、不规定主题，它都反复使用 `孤舟`、`雁字`、`芦花`、`暮云`、`荻花`、`长亭`。

它的古典化程度比较高，但也最容易滑向熟悉的“秋江送别式”模板。

### 3.4 一旦给定主题，不能不把主题原文写出来

这一部分如果不把主题原文写出来，读者其实无法判断后面的意象收缩到底收缩到了哪里。

改革主题原文是：

> 假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。

玉兰主题原文是：

> 假设你是一个喜欢写宋词的大学生，最近在学业和面试上屡遇挫折，但是你看到了一颗盛开着白玉兰花的树，似乎重新找到了生命的实在与价值，便写了一首词。

只有把这两段原文摆出来，后面“为什么会出现樊笼、灵根、晨钟”或者“为什么会出现玉兰、素萼、寒窗”才真正说得清。

### 3.5 改革主题会把意象池拉向什么方向

在改革主题 prompt 下，高频意象非常稳定地变成了宗门化、制度化、修仙化词汇。

五词牌 + 改革主题时，最常出现的是：

- 樊笼
- 劫火
- 灵根
- 剑气
- 铁律
- 青衿
- 星斗

固定 `沁园春` + 改革主题 + 简略格律提醒时，则进一步集中到：

- 青衿
- 灵根
- 樊笼
- 丹炉
- 晨钟
- 暮鼓
- 仙峰
- 弟子

也就是说，模型并没有把“改革”写成开放社会议题，而是迅速把它吸收到一套“宗门旧规、危机、战力、秩序、修行”的意象框架中。

### 3.6 玉兰主题会把意象池拉向什么方向

玉兰主题的收缩更强，也更单一。

五词牌 + 玉兰主题时，高频意象是：

- 玉兰
- 寒窗
- 素萼
- 玉树
- 暗香
- 春风

固定 `沁园春` + 玉兰主题 + 显式模板时，则集中为：

- 玉兰
- 琼枝
- 寒窗
- 琼英
- 玉树
- 清芬

这说明玉兰主题几乎天然会把模型压到一条很固定的写法上：先写花之洁白清芬，再写受挫与寒窗，最后写由花而振作。

### 3.7 本部分结论

这一部分真正重要的不是“AI 常用哪些词”，而是：

1. 弱约束 prompt 下，AI 已经会自动回到一套成熟的宋词意象库。
2. 词牌会继续改变意象组合方式。
3. 不同模型的默认意象池并不相同。
4. 一旦给定主题，意象会急速收缩；而且如果不把主题原文写出来，这种收缩其实是说不清的。

## 四、AI 实际在写什么主题

### 4.1 这一部分的主题不是无限开放的，而是在候选类型中归类

本项目里的“主题分类”不是完全开放生成，而是把作品归入若干候选主题类型。  
在无主题或弱主题任务中，主要出现的候选主题一共九类：

1. 离别羁旅
2. 闲适隐逸
3. 豪情壮志
4. 悼亡相思
5. 咏物寄托
6. 谈禅说理
7. 怀古咏史
8. 山水田园
9. 春怨秋悲

因此，第四部分不能只说“AI 写成了什么主题”，还必须先告诉读者：它其实是在这些候选类型里不断滑向其中几类。

### 4.2 无主题 prompt 并不等于“真的没有主题”

这一点不能只看固定五词牌任务，还要把完全自由生成和固定五词牌、不规定主题一起看。

完全自由生成时，占比最高的是：

- 离别羁旅：`34.3%`
- 闲适隐逸：`14.7%`
- 豪情壮志：`10.2%`
- 悼亡相思：`9.2%`
- 咏物寄托：`9.0%`

固定五词牌、不规定主题时，收缩更明显：

- 离别羁旅：`42.7%`
- 闲适隐逸：`15.7%`
- 怀古咏史：`9.8%`
- 悼亡相思：`7.6%`
- 豪情壮志：`7.1%`

这说明即使不规定主题，AI 也不会真的自由发散，而是会稳定滑向几类自己最熟悉、也最容易用传统意象支撑的题材，尤其是离别羁旅。

### 4.3 不同模型在无主题 prompt 下的默认题材也不同

无主题任务里，模型差异依然很明显。

完全自由生成时：

- `doubao` 更偏 `闲适隐逸`、`山水田园`、`离别羁旅`。
- `deepseek` 更偏 `咏物寄托`、`悼亡相思`、`豪情壮志`、`谈禅说理`。
- `qwen3.6-plus` 则极强地集中在 `离别羁旅`，比例高达 `59.2%`。

固定五词牌、不规定主题时：

- `doubao`：`离别羁旅` 占 `48.0%`，还保留较多 `闲适隐逸` 和 `怀古咏史`。
- `deepseek`：`离别羁旅` 只占 `28.0%`，`豪情壮志` 与 `谈禅说理`明显更高。
- `qwen3.6-plus`：`离别羁旅` 高达 `57.4%`，是三家里最集中地落在这一类上的模型。

所以，第四部分不能只讲“AI 会写什么主题”，而应讲“不同模型默认会把题材组织到哪里去”。

### 4.4 改革主题最后被压缩成什么

先把改革主题原文再放一次，因为后面的分析必须以它为依据：

> 假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。

这个主题看上去叙事很具体，但模型最后主要压缩成三类：

在“五词牌 + 改革主题”中：

- 谈禅说理：`41.1%`
- 咏物寄托：`39.6%`
- 豪情壮志：`19.3%`

在“固定 `沁园春` + 改革主题 + 简略格律提醒”中：

- 谈禅说理：`57.2%`
- 豪情壮志：`32.6%`
- 咏物寄托：`9.8%`

模型级差异也很明显：

- `doubao` 在改革主题里最容易写成 `谈禅说理`。
- `deepseek` 在五词牌改革主题里 `谈禅说理` 与 `咏物寄托` 接近对半。
- `qwen3.6-plus` 在固定 `沁园春` + 改革主题里反而更容易落到 `豪情壮志`。

这说明改革主题并没有被模型展开成复杂叙事，而是被翻译成“说理”“寄托”“壮志”这三条熟悉路线。

### 4.5 玉兰主题最后被压缩成什么

玉兰主题原文同样必须保留：

> 假设你是一个喜欢写宋词的大学生，最近在学业和面试上屡遇挫折，但是你看到了一颗盛开着白玉兰花的树，似乎重新找到了生命的实在与价值，便写了一首词。

这个主题最终几乎稳定压缩成 `咏物寄托`：

- 五词牌 + 玉兰主题：`咏物寄托` 占 `92.0%`
- 固定 `沁园春` + 玉兰主题 + 显式模板：`咏物寄托` 占 `87.7%`

模型级差异虽然存在，但总趋势高度一致：

- `doubao`：`咏物寄托` 占 `91.5%` 或 `84.0%`
- `deepseek`：在 `c1` 里甚至高到 `96.5%`
- `qwen3.6-plus`：也分别达到 `91.5%` 与 `81.2%`

这说明玉兰主题不是“写一朵花”这么简单，而是几乎被稳定翻译成“借花写人、借花写受挫后的重振”。

### 4.6 本部分结论

第四部分需要明确四点：

1. AI 的主题归类不是无限开放，而是在若干候选类型中滑动。
2. 无主题 prompt 并不真的没有主题，模型会自动回到自己熟悉的题材。
3. 改革主题主要被压缩成“谈禅说理、咏物寄托、豪情壮志”。
4. 玉兰主题几乎被稳定翻译成“咏物寄托”，这是项目里最强的主题收缩之一。

## 五、AI 宋词是否借鉴真实宋词

### 5.1 分析逻辑：先建立历史基线，再谈借用关系

这一部分不能直接上“抄袭”二字。更稳妥的逻辑是三步：

1. 先看真实宋词内部本来就有多像。
2. 再看 AI 整首最接近哪些真实作品。
3. 最后看 AI 的多个句子是否反复回到同一首真实来源。

只有先建立历史基线，后面的“借用关系”才不会说得过重。

### 5.2 数据规模与历史基线

本轮相似性分析覆盖：

- AI 作品 `3122` 首
- AI 句子 `49304` 句
- 真实宋词 `21053` 首
- 真实句子 `270107` 句

真实宋词内部的相似度本来就不低。加权汇总后：

- 整首语义相似均值约 `0.8404`
- 整首字面相似均值约 `0.2366`
- 句级语义相似均值约 `0.7318`
- 句级字面相似均值约 `0.4415`

更关键的是，AI 与真实宋词比对中一共出现了 `2145` 条句级字面 `1.0` 的完全匹配，但其中：

- `2127` 条来自 `<=5` 字短句
- `18` 条来自 `6-8` 字句
- `9` 字以上长句几乎没有完全一致

这说明大量完全一致集中在短句和套语级，不能把所有重合一概视为异常借用。

### 5.3 整体判断：大多数作品更像“传统资源池复用”，而不是整首重写

从整首级结果看：

- 整首语义分位进入历史 `P95` 以上的作品：`45` 首，占 `1.44%`
- 整首字面分位进入历史 `P95` 以上的作品：`46` 首，占 `1.47%`
- 语义主来源覆盖率 `>= 0.2` 的作品：`30` 首，占 `0.96%`
- 字面主来源覆盖率 `>= 0.2` 的作品：`41` 首，占 `1.31%`
- 主来源覆盖率 `>= 0.5` 的作品：语义和字面都是 `0`

这意味着在全部 `3122` 首 AI 作品中，几乎没有哪一首可以说“半数以上句子都回到了同一首真实宋词”。  
多数样本更像是从传统宋词资源池里分散调用意境、局部套语和成熟抒情骨架。

### 5.4 强约束不一定更危险，长调高相似也不等于异常

按任务条件看，最值得提醒读者的是：

| 任务说明 | 平均整首语义分 | 平均语义分位 | 平均整首字面分 | 平均字面分位 |
|---|---:|---:|---:|---:|
| 完全自由生成 | 0.8539 | 0.6916 | 0.1991 | 0.3376 |
| 五词牌、无主题 | 0.8524 | 0.6326 | 0.2099 | 0.3640 |
| 五词牌、改革主题 | 0.8352 | 0.5040 | 0.1880 | 0.1476 |
| 五词牌、玉兰主题 | 0.8477 | 0.6032 | 0.1938 | 0.2004 |
| 固定 `沁园春`、玉兰主题、显式模板 | 0.8723 | 0.2453 | 0.1792 | 0.1694 |
| 固定 `沁园春`、改革主题、简略提醒 | 0.8695 | 0.2053 | 0.1788 | 0.1289 |

这里最关键的是：

1. `c1/c2` 的整首语义分很高，但分位反而很低。  
这说明固定 `沁园春` 的长调任务确实更容易写出“很像传统长调”的文本，但这更多是因为 `沁园春` 本身历史基线就很高，而不一定是异常借用。

2. 真正更容易碰到历史高尾部的，反而常常是自由或半自由条件。  
因为这类任务更容易直接滑入成熟词境模板。

### 5.5 哪些词牌更敏感

在“五词牌、不规定主题”的对照组里，五个词牌的风险形态不同：

- `祝英台近`：句级语义 `P95` 比例 `56.2%`，句级字面 `P95` 比例 `70.1%`，最容易出现局部借用与局部改写。
- `浪淘沙`：整首语义分位均值 `0.7627`，更像整体落入熟悉意境。
- `沁园春`：整首字面最近邻回到同词牌的比例 `32.4%`，同词牌吸附最明显，但不能离开其高历史基线单独解读。
- `清平乐` 与 `菩萨蛮`：更接近常规模板化，而不是强烈的局部借用敏感。

### 5.6 具体例子：AI 宋词到底借了谁

这一部分如果没有具体例子，读者其实很难真正理解“借用关系”。

#### 例一：高语义贴近，但来源分散，不像整首重写

样本：`a1:81`，`qwen3-max`，`青玉案《秋江夜泊》`

AI 开头是：

> 西风卷尽芦花雪，又送客、孤舟发。月照寒汀烟水阔。

这首词整首语义分位达到 `0.9781`，最近邻整体最像方岳的《齐天乐》，但主来源覆盖率只有 `0.0833`。  
也就是说，它不像在重写某一首词，更像是把传统离愁羁旅词中已经非常成熟的一组构件重新拼装了一遍。

#### 例二：局部句法骨架开始回到同一首来源

样本：`b2:364`，`deepseek-3.2-thinking`，`祝英台近《仙门感怀》`

AI 文本中有这样几句：

> 怕上层楼，危机暗中满。  
> 断肠规矩如链，青春空晚，更谁劝、莺声留挽？

它的整首字面分位达到 `1.0000`，字面主来源覆盖率 `0.25`，最近邻回到辛弃疾《祝英台令》。  
其中多处关系已经很明显：

- `怕上层楼` 对 `怕上层楼`
- `更谁劝、莺声留挽` 对 `倩谁唤、流莺声住`

这种情况已经不是一般“传统共相”，而是开始接近局部改写式借用。

#### 例三：长调强约束下回到苏轼《沁园春》

样本：`c2:193`，`deepseek-3.2-thinking`，`沁园春《宗规感怀》`

AI 文本中出现了：

> 世路艰险，劳生有限，似此压抑长难容。微吟罢，凭栏杆无语，心事重重。  
> 有宏图千卷，胸中万策；致君尧舜，此事非空。  
> 用舍由时，行藏在我……

它的整首字面分位达到 `0.9905`，语义主来源覆盖率 `0.36`，字面主来源覆盖率 `0.32`，最近邻显著回到苏轼《沁园春》。  
这里的关系已经非常直接：

- `劳生有限` 对 `劳生有限`
- `微吟罢` 对 `微吟罢`
- `胸中万策` 对 `胸中万卷`
- `致君尧舜` 对 `致君尧舜`
- `用舍由时，行藏在我` 对 `用舍由时，行藏在我`

这类案例最值得人工复核，因为它已经不只是“像一类词”，而是在关键结构节点上持续滑向某一首名作。

### 5.7 模型差异：高风险尾部主要聚集在 `deepseek`

如果把“高风险”暂时定义为：

- 整首字面分位进入历史 `P95`
- 且字面主来源覆盖率 `>= 0.2`

那么全项目只有 `15` 首符合，而这 `15` 首全部来自 `deepseek-3.2-thinking`。它们又主要集中在：

- 固定 `沁园春` + 改革主题 + 简略提醒：`6` 首
- 五词牌 + 改革主题：`5` 首
- 五词牌、不规定主题：`4` 首

按词牌看，又主要集中于：

- `沁园春`：`11` 首
- `祝英台近`：`4` 首

所以，这一部分不能只写“AI 会不会借用真实宋词”，而应更准确地写成：**借用风险并不均匀分布，它非常依赖模型，也依赖具体任务设置。**

### 5.8 本部分结论

第五部分应当落在四个判断上：

1. 绝大多数 AI 宋词并不呈现“整首单一来源重写”。
2. 更常见的是传统资源池、短句套语和成熟词境的高密度复用。
3. 真正值得警惕的是局部来源聚集，尤其是 `deepseek` 在 `祝英台近` 和 `沁园春` 上的部分样本。
4. 因此，这里最合适的表述是“借鉴”“借用关系”“真实来源相似性”，而不是不加区分地说“抄袭”。

## 六、AI 宋词是否同质化

### 6.1 先用具体例子讲清楚：什么叫字面相似，什么叫语义相似

这一部分如果只讲“余弦相似度”和“字面匹配”，读者会很抽象，所以必须给例子。

#### 语义相似的例子

例如：

- AI 句子：`素萼破寒轻绽`
- 真实来源：`绛萼欺寒`

这组句子的语义相似度约 `0.9001`，但字面相似度只有 `0.4`。  
它们的字面并不接近，但都在写“花萼在寒意中开放”的同一类语义结构。

再如：

- AI 句子：`几度风萍浪梗`
- 真实来源：`此际浪萍风梗`

语义相似度约 `0.9045`，字面相似度约 `0.4444`。  
这里词序、用字都变了，但整体表达方向仍然高度一致。

这就是语义相似：**换了词，却没有换掉句子的表达骨架。**

#### 字面相似的例子

例如：

- AI 句子：`银汉无声转玉盘`
- 真实来源：`银汗无声转玉盘`

这组句子的字面相似度约 `0.8571`，语义相似度约 `0.9003`。  
它们几乎只差一个字，属于非常强的局部贴近。

再如：

- AI 句子：`暗香浮动黄昏后`
- 真实来源：`暗香浮动黄昏后`

这是字面与语义都完全一致的典型例子。

因此，语义相似和字面相似不能混为一谈：

- 语义相似更偏“换词不换意”。
- 字面相似更偏“这句话本身是不是已经很接近原句”。

### 6.2 中文术语必须说清楚

为了避免报告里一直出现缩写，这里统一把几个核心指标翻成中文：

- `rhythm_score`：格律分
- `whole_score`：整词同质化度
- `sent_score`：句子同质化度
- `imagery_score`：意象同质化度
- `core_mihi`：核心同质化度
- `mihi_full`：总同质化度
- `excess_core_homogeneity`：超额核心同质化度

后文都尽量以中文术语为主，只在必要时保留括号中的原始字段名。

### 6.3 这一部分最关键的判断：整词同质化在主题任务下是正常的，句子同质化才真正危险

这是第六部分最需要重新审视、也最需要说清楚的地方。

同题写作本来就应该在整首层面更接近。  
如果几百首“改革主题”或“玉兰主题”的词，在整体情境、主导情绪、意象方向上完全不接近，那反而说明 prompt 没有起作用。

因此，**整词同质化度上升，在主题约束下是正常现象，也是应该发生的现象。**  
真正需要警惕的是：句子同质化度是不是也被一并推高，导致局部表达开始模板化、套语化。

### 6.4 用“整词同质化度 - 句子同质化度”的差值来理解这个问题

如果把整词同质化度减去句子同质化度，可以更直观地分辨：模型是在“同题写作下整体靠近”，还是已经“局部句子也越来越像”。

这里先给一个简化判读标准。这个差值不是总同质化程度本身，而是**句子层模板化严重性**的一个快速指示：

- 差值 `<= 0.18`：句子层已经追得很近，局部模板化风险偏重。
- 差值在 `0.18 - 0.30`：主要是整词层收束，句子层风险中等。
- 差值 `>= 0.30`：整词收束明显强于句子收束，更多是“同题写作很像”，句子层相对没那么紧。

按三家主模型平均后，大致是：

| 任务说明 | 整词同质化度均值 | 句子同质化度均值 | 二者差值 | 用差值做的快速判断 |
|---|---:|---:|---:|---|
| 五词牌、无主题 | 0.6501 | 0.4953 | 0.1548 | 句子层风险偏重 |
| 五词牌、改革主题 | 0.7263 | 0.4384 | 0.2880 | 中等，主要是整词收束 |
| 五词牌、玉兰主题 | 0.7104 | 0.4728 | 0.2377 | 中等，主要是整词收束 |
| 固定 `沁园春`、玉兰主题、显式模板 | 0.8951 | 0.5362 | 0.3589 | 整词收束很强，句子层相对次之 |
| 固定 `沁园春`、改革主题、简略提醒 | 0.9256 | 0.5082 | 0.4174 | 整词收束极强，句子层相对次之 |

这张表非常重要，因为它说明：

1. 一旦给定主题，整词同质化度会明显上升。
2. 但在“五词牌 + 主题”这一步，差值反而变大了。
3. 这说明主题首先压缩的是“整首词写什么”，而不是“每一句都写得越来越像”。

换句话说，**主题 prompt 先带来的是合理的整词收束，而不是坏的句子模板化。**

如果继续往下拆到“同样主题内部，不同模型从五词牌条件走向 `沁园春` 强约束时，差值本身和差值变化怎么变”，情况会更具体。

#### 改革主题内部：`b2 -> c2` 的模型差值

| 模型 | `b2` 差值 | `c2` 差值 | 差值变化 | 用差值做的快速判断 | 解释 |
|---|---:|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `0.2711` | `0.3579` | `+0.0868` | 从中等转向整词收束更强 | 整词与句子都不低，但进入 `c2` 后整词抬得更快，说明它首先在整体结构和主题框架上进一步收紧 |
| `deepseek-3.2-thinking` | `0.2800` | `0.4264` | `+0.1464` | 从中等转向整词收束极强 | 这里的 `c2` 取观察分析低样本值；差值变大说明 surviving 样本更多集中在整词层，但也意味着高格律可用样本太少 |
| `qwen3.6-plus` | `0.3128` | `0.4679` | `+0.1551` | 本来就偏整词收束，进入 `c2` 后更明显 | 句子层没有同步追高，更多是整词层继续收紧，而不是局部模板突然恶化 |

这里需要单独说明：`deepseek-3.2-thinking` 与 `qwen3.6-plus` 在 `c2` 的严格主分析里都因为高格律样本不足，只能引用观察分析值。因此，`c2` 不能只当作一个“同质化变化”的问题来看，它首先还是一个显著的形式失稳问题。

#### 玉兰主题内部：`b3 -> c1` 的模型差值

| 模型 | `b3` 差值 | `c1` 差值 | 差值变化 | 用差值做的快速判断 | 解释 |
|---|---:|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `0.2450` | `0.3552` | `+0.1102` | 从中等转向整词收束更强 | `c1` 中句子层也高，但整词层抬得更快，说明显式模板先把整体结构做得更整齐 |
| `deepseek-3.2-thinking` | `0.2294` | `0.3547` | `+0.1253` | 从中等转向整词收束更强 | 进入 `c1` 后，整词层被显著拉紧，句子层也升，但没有追得和整词一样快 |
| `qwen3.6-plus` | `0.2386` | `0.3666` | `+0.1280` | 从中等转向整词收束更强 | 它在 `b3` 时已经有稳定主题收束，到了 `c1` 后更多是整词进一步靠近，而不是句子层突然爆发 |

把这两张表和前面的总体表放在一起，就能得到一个更清楚的判断：**同样是更强约束，不同模型虽然都会把差值拉大，但“拉大”的含义并不一样。有的模型是在整词层继续收紧，有的模型则是在高格律筛选后只剩下少量很像的样本。**

### 6.5 判断严重性时，不能只看差值变化，还要看差值本身落在什么区间

现在可以把这一部分的逻辑统一成三步：

1. 先看差值本身：判断当前更像“整词收束”还是“句子模板化”。
2. 再看差值变化：判断强约束之后压力是在向整词层移动，还是向句子层移动。
3. 最后再看句子同质化度本身：判断局部句法是否已经高到值得警惕。

因此，真正的风险并不是“只要同质化升高就危险”，而是下面两种情况：

- 差值本身偏小，说明句子层已经和整词层靠得很近。
- 或者差值虽然不小，但句子同质化度本身仍然很高，说明模型在整体很像的同时，局部表达也已经明显收紧。

从当前数据看：

- `b1` 的平均差值只有 `0.1548`，这反而说明无主题条件下，模型更容易直接滑向熟悉句法和套语。
- `b2` 与 `b3` 的差值扩大到 `0.2880` 和 `0.2377`，说明主题先把压力集中到整词层，这是相对正常的。
- `c1` 与 `c2` 的差值进一步扩大到 `0.3589` 与 `0.4174`，说明强约束后整词收束极强；但与此同时，`c1` 的句子同质化度均值仍高达 `0.5362`，`c2` 也达到 `0.5082`，所以不能简单把“大差值”理解成“完全不危险”。

更准确地说：

- 小差值：更像“句子层已经危险地跟上来了”。
- 大差值：更像“整体写得太像了”，但还要继续检查句子分是不是也已经过高。

### 6.6 主题效应是正常的，但模型变量决定了“危险到底落在哪一层”

如果沿着上面的三步逻辑继续往下看，就会发现：真正决定风险分布的，更多还是模型变量，而不是任务名字本身。

#### `doubao-seed-2.0-pro`

`doubao` 的特点是：格律最好，整词层也最稳。  
从差值看，它在 `b1` 时只有 `0.1395`，句子层风险已经不低；到了 `b2`、`b3` 后分别扩大到 `0.2711`、`0.2450`，说明主题先把它拉回整词收束；进入 `c1`、`c2` 后又升到 `0.3552`、`0.3579`，说明它最容易把强约束任务做成“整体极稳、整体极像”的文本。

因此，`doubao` 的风险画像不是“最容易失控”，而是**最容易把任务做得整齐，也最容易因此落入高强度的整体收束**。

#### `deepseek-3.2-thinking`

`deepseek` 的特点是：主题牵引最强，来源风险也最集中。  
它从 `b1` 的 `0.1416` 到 `b2` 的 `0.2800`、`b3` 的 `0.2294`，说明一加主题就会明显从句子层模板转向整词层收束；到 `c1` 与观察分析下的 `c2`，差值又升到 `0.3547` 和 `0.4264`。  
但这并不意味着它“最安全”，因为真实来源分析里高风险尾部样本几乎都集中在它身上。

因此，`deepseek` 的画像是：**最容易被主题牵着走，整词层和意象层收缩明显，同时局部来源风险也最值得单独盯防。**

#### `qwen3.6-plus`

`qwen3.6-plus` 的特点是：无主题时就偏集中，长调时又容易失稳。  
它在 `b1` 的差值已到 `0.1833`，说明默认写法就比较集中；到 `b2`、`b3` 时差值变成 `0.3128`、`0.2386`；进入 `c1` 与观察分析下的 `c2` 后又升到 `0.3666`、`0.4679`。  
这说明它并不是在句子层不断加剧模板化，而更像是在整体题材和整体情境上不断收缩。

因此，`qwen` 的画像是：**默认就有较强模板化倾向，进入长调后又容易因形式失稳而只留下少量高度集中的样本。**

所以，同质化问题不能只讲“哪个任务更高”，而应统一成一句话：**同一个任务会把不同模型推向不同层面的收缩，有的主要收缩在整词层，有的会连句子层和来源风险一起带高。**

### 6.7 本部分结论

第六部分应当落在六个判断上：

1. 这一部分不能只看单一相似度，而必须同时区分语义相似与字面相似，再把整词层和句子层分开讨论；否则很容易把“同题写作的正常接近”误判成“局部模板化”。
2. 判断同质化时，第一步要先看“整词同质化度 - 句子同质化度”的差值本身。差值偏小，说明句子层已经追得很近；差值偏大，说明当前更像是整词层先被主题和结构收紧。
3. 判断同质化时，第二步要再看差值变化。`b1 -> b2` 与 `b1 -> b3` 中差值扩大，说明主题先压缩的是整首词的内容方向；`b2 -> c2` 与 `b3 -> c1` 中差值继续扩大，则说明强约束进一步强化了整体收束。
4. 因此，主题任务下整词同质化度上升本身并不必然是坏事，它首先说明 prompt 生效了。真正需要警惕的，是句子同质化度也升得很高，或者差值虽然不小，但句子分本身已经维持在高位。
5. 三个主模型的风险层次并不一样：`doubao` 更容易把任务做得极稳，从而落入高强度的整体收束；`deepseek` 最容易被主题牵引，整词、意象和来源风险会一起集中；`qwen` 在无主题条件下已有较强默认模板化倾向，进入长调后又容易因为形式失稳而只剩少量高度集中的样本。
6. 所以，第六部分的最终判断不应再写成“哪个任务同质化更高”这么简单，而应写成：**同质化要按“差值本身 + 差值变化 + 模型画像”三步来读，只有这样才能区分正常的主题收束、值得警惕的句子模板化，以及不同模型各自的风险来源。**

## 七、这些发现对之后使用 AI 写宋词有什么启发

### 7.1 如果目标是“尽量稳地写对”

建议做法是：

- 明确给定词牌，不要完全自由放开。
- 遇到长调，尤其是 `沁园春`，最好提供显式模板，而不要只说“请注意格律”。
- 生成之后一定要接程序化格律校验，不能只凭人工目测。

### 7.2 如果目标是“尽量多样，不要都写成一类”

建议做法是：

- 不要把词牌、主题、长调、强风格提示一次性全叠满。
- 可以只固定词牌、放开主题，或只固定主题、放开词牌。
- 批量生成后再筛选，比一次性追求“单首直接完美”更容易保留变化空间。

### 7.3 如果目标是“减少模板化”

建议做法是：

- 不要只看单首输出，而要比较同一模型在同一任务下批量生成的多首作品。
- 不能只看整词同质化度，还要同时看句子同质化度。
- 可以把“整词同质化度 - 句子同质化度”的差值作为一个实用观察指标：整词收束是正常的，句子层大幅抬高才值得警惕。

### 7.4 如果目标是“降低与真实宋词过近的风险”

建议做法是：

- 对 `祝英台近` 和 `沁园春` 这类更敏感的词牌单独警惕。
- 对句子层高度贴近的样本做人工复核，尤其优先检查较长句。
- 对 `deepseek` 在改革主题与长调任务中的结果增加额外复核，因为高风险尾部样本目前主要聚集在这里。
- 在正式生成流程里加入真实来源比对模块，而不是只在事后抽查。

### 7.5 最终总结

综合来看，大模型已经具备了一定的宋词生成能力，但这种能力高度不均衡：它们能在不少场合写出形式基本成立、意境也像样的作品，却也会在长调、强主题和批量生成中迅速暴露出平仄失守、题材收缩、意象固化和局部借用风险；因此，真正可用的工作流不应是“把 AI 当成会写词的人”，而应是“把 AI 当成一个需要在明确约束、自动校验和人工复核中使用的写作引擎”。
