# 相似性分析方法论详解

本文档详细描述了AI宋词相似性分析的方法设计、问题诊断与优化过程。

## 一、相似度计算方法

### 1.1 语义相似度 (BERT)

**计算方式**：
```python
# 使用BERT-CCPoem模型将文本转换为向量
embeddings = model.encode(texts, convert_to_tensor=True)
# 计算余弦相似度矩阵
cos_matrix = util.cos_sim(embeddings, embeddings)
```

**特点**：
- 衡量意境、情感、主题的相似性
- 对"换词不换意"敏感
- 范围：0-1，越高越相似

### 1.2 字面相似度 (difflib)

**计算方式**：
```python
def calc_lexical_similarity(text1, text2):
    return difflib.SequenceMatcher(None, text1, text2).ratio()
```

**特点**：
- 衡量字词、句式的重复程度
- 基于最长公共子序列
- 范围：0-1，越高越重复

### 1.3 两者结合的意义

| 组合情况 | 语义 | 字面 | 差值 | 含义 |
|---------|------|------|------|------|
| 高语义 + 高字面 | 高 | 高 | 小 | 直接重复/抄袭 |
| 高语义 + 低字面 | 高 | 低 | 大 | **高级伪原创** |
| 低语义 + 高字面 | 低 | 高 | 负 | 换意不换词（罕见） |
| 低语义 + 低字面 | 低 | 低 | 小 | 完全不同 |

**伪原创比例**：差值 > 0.3 的句对比例，反映"换词不换意"的严重程度。

## 二、统计指标设计

### 2.1 分位数统计

使用P25/P50/P75/P90/P95代替简单平均：

| 指标 | 说明 | 优势 |
|------|------|------|
| P50(中位数) | 一半数据低于此值 | 抗极端值影响 |
| P90 | 90%数据低于此值 | 反映严重重复程度 |
| P95 | 95%数据低于此值 | 最极端情况 |

**为什么不用平均数？**
- 平均数受极端值影响大
- 无法反映分布形态
- 分位数能更全面描述分布

**问题诊断**：

实际数据中熵值约0.70-0.76，反向熵约0.22-0.26，所有模型差异很小。

**原因分析**：
- 熵衡量的是分布的"形状"，不是"位置"
- 当所有模型的相似度都集中在0.3-0.5区间时，熵值都差不多
- 熵无法区分"相似度集中在0.3-0.4"和"相似度集中在0.7-0.8"

**结论**：熵不适合作为相似度的衡量指标，已从综合评分中移除。

### 2.3 高相似比例

**问题**：原阈值0.85太高，导致比例只有0.1%-0.5%，贡献几乎为0。

**计算示例**：
```
高语义比例贡献 = 0.25 × 0.0025 = 0.0006 ← 几乎为0
```

**结论**：高相似比例指标在当前数据分布下意义有限，已从综合评分中移除。

## 三、综合评分公式演进

### 3.1 初始版本

```
综合评分 = 0.30 × P50 + 0.25 × P90 + 0.25 × 高相似比例 + 0.20 × 熵(反向)
```

**问题**：
- 高相似比例贡献几乎为0
- 熵区分度不够
- 综合评分范围0.26-0.29，差异太小

### 3.2 中间版本（加入字面相似度）

```
综合评分 = 0.20 × P50 + 0.15 × P90 + 0.15 × 高相似比例 + 0.10 × 熵(反向) 
         + 0.20 × 字面P50 + 0.20 × 伪原创比例
```

**问题**：
- 字面相似度与语义相似度量纲不同
- 伪原创比例与语义P50高度相关，存在冗余
- 用户反馈：不需要将字面相似度纳入综合评分

### 3.3 最终版本

```
综合评分 = 0.55 × P50 + 0.45 × P90
```

**优势**：
- 简单直接，易于理解
- P50和P90差异能充分体现
- 综合评分范围约0.35-0.50，区分度好

## 四、分析粒度对比

### 4.1 句级别分析 (pytorch-analysis-3/4)

**计算方式**：
- 将每首词拆分为句子
- 计算跨词句对的相似度
- 统计所有跨词句对的分布

**特点**：
- 粒度细，能发现局部重复
- 对单句换词不换意敏感
- 数据量大（n句 → n(n-1)/2句对）

### 4.2 词级别分析 (pytorch-analysis-2)

**计算方式**：
- 整首词作为一个整体计算embedding
- 计算词与词之间的相似度
- 统计所有词对的分布

**特点**：
- 粒度粗，发现整体风格相似
- 对整体结构/风格相似敏感
- 数据量小（m词 → m(m-1)/2词对）

### 4.3 两者对比

| 维度 | 句级别 | 词级别 |
|------|-------|-------|
| 敏感度 | 对单句重复敏感 | 对整体风格相似敏感 |
| 数据量 | 大 | 小 |
| 用途 | 检测换词不换意的句子 | 检测整体风格固化 |
| 可能漏掉 | 整体风格相似 | 局部句子重复 |

## 五、融合分析方案

### 5.1 双维度诊断矩阵

```
                    词级别评分
                    低 ←────────→ 高
              ┌─────────┬─────────┐
         低   │  多样性  │  风格一致 │
句级别评分     │   良好   │  表达多样 │ ← 好的情况
         ↓    ├─────────┼─────────┤
         高   │  局部重复 │  整体同质 │
              │   问题   │   化严重  │ ← 问题严重
              └─────────┴─────────┘
```

### 5.2 四种诊断结果

| 类型 | 词级别 | 句级别 | 诊断 | 问题程度 |
|------|--------|--------|------|---------|
| A | 低 | 低 | 多样性良好 | ✅ 无问题 |
| B | 高 | 低 | 风格一致但表达多样 | ✅ 好的倾向 |
| C | 低 | 高 | 局部重复问题 | ⚠️ 需关注 |
| D | 高 | 高 | 整体同质化严重 | ⚠️⚠️ 最严重 |

### 5.3 融合指标

| 指标 | 公式 | 含义 |
|------|------|------|
| 整体同质化指数 | 词P50 × 句P50 | 两者都高时指数高 |
| 局部重复指数 | 句P90 - 词P90 | 句级别明显高于词级别时，存在局部重复 |
| 风格一致性指数 | 词P50 - 句P50 | 词级别明显高于句级别时，风格一致但表达多样 |
| 融合诊断评分 | 0.4 × 词评分 + 0.6 × 句评分 | 综合同质化评分 |

### 5.4 融合优势

- **全面诊断**：不漏掉整体风格或局部重复问题
- **问题定位**：能区分问题类型（整体vs局部）
- **评分更准确**：融合评分比单一评分更可靠
- **可视化直观**：散点图一目了然看出问题类型

## 六、可视化设计

### 6.1 箱线图

**用途**：直观展示各模型相似度的整体分布

**解读**：
- 箱体位置越高 = 同质化越严重
- 箱体越窄 = 相似度集中，风格一致
- 箱体越宽 = 多样性好

### 6.2 热力图

**用途**：快速定位哪些模型+词牌组合同质化最严重

**颜色编码**：
- 绿色系：低相似度（多样性好）
- 黄色系：中等相似度
- 红色系：高相似度（同质化严重）

### 6.3 KDE密度曲线

**用途**：精细展示相似度的概率密度分布

**曲线特征解读**：
- 峰值位置靠右 = 大部分句对高相似，同质化严重
- 峰值位置靠左 = 大部分句对低相似，多样性好
- 单峰窄分布 = 风格一致，相似度集中
- 多峰分布 = 存在多个相似模式

### 6.4 雷达图

**用途**：综合展示各模型的多项指标

**维度**：语义P50、语义P90、综合评分

## 七、性能优化

### 7.1 内存优化

**问题**：进度到99%时，程序突然增加内存需求直到占满

**原因**：`all_pairs`列表存储所有句对的完整数据

**解决方案**：使用堆(heap)只保留Top N高相似句对

```python
# 原版：存储所有句对
all_pairs = []
for pair in all_pairs_generator():
    all_pairs.append(pair)  # O(n²) 内存

# 优化版：只保留Top N
import heapq
top_heap = []
for pair in all_pairs_generator():
    if len(top_heap) < N:
        heapq.heappush(top_heap, pair)
    elif pair > top_heap[0]:
        heapq.heapreplace(top_heap, pair)  # O(N) 内存
```

**效果**：内存减少99%+
