## 一、项目核心探讨的问题

### 1.1 核心问题

这个项目研究的，不是AI写的宋词好不好，而是更具体的五个问题：

1. AI 能不能写出在格律和结构上基本成立的宋词。
2. 在不同任务条件下，AI 会把词写成什么样。
3. AI 会稳定偏爱哪些意象、哪些题材、哪些写法。
4. AI 在批量生成时会不会越来越写成固定模板。
5. AI 生成的宋词与真实宋词之间，到底是风格接近和化用，还是出现了更值得警惕的局部借用亦或是照抄。

换句话说，这个项目不是只看“像不像宋词”，而是同时看形式能力、内容组织、内部同质化和与真实作品的关系。

### 1.2 整个项目的流程

整个项目的执行流程可概括为六个步骤：

1. 在 `generate/prompts.py` 中定义 Prompt（提示词）模板。
2. 在 `generate/json/*.json` 中设定调用的模型、数据库、词牌池和主题文本。
3. 批量调用模型生成 JSON 格式的宋词，并将其写入 `database/*.db`。
4. 使用 `engine/songci_engine.py` 对字数、平仄、押韵进行自动化校验。
5. 为每首词抽取意象并进行主题类别归类，形成描述性统计。
6. 进一步开展两类深度分析：一类是 AI 与真实宋词的来源相似性分析，另一类是 AI 生成样本自身的内部同质化分析。

### 1.3 六类任务的具体设定

本项目设计了六组测试任务，这些任务并非随机分组，而是构成了一条约束条件逐步递进的测试链：

| 数据库 | 中文解释 | 任务性质 | Prompt 类型 |
|---|---|---|---|
| `a1` | 自由生成 | 不限词牌，不限主题 | `type_c` |
| `b1` | 五词牌、无主题 | 仅添加词牌约束 | `type_b` |
| `b2` | 五词牌、改革主题 | 词牌约束 + 主题约束 | `type_d` |
| `b3` | 五词牌、玉兰主题 | 词牌约束 + 主题约束 | `type_d` |
| `c1` | 固定 `沁园春`、玉兰主题、显式格律模板 | 强结构约束任务 | `type_a` |
| `c2` | 固定 `沁园春`、改革主题、简略格律要求 | 强主题 + 简略格律提示 | `type_e` |

上面的五词牌指：`清平乐` `浪淘沙` `菩萨蛮` `沁园春` `祝英台近`，生成时随机选择一个词牌。
此处必须明确界定：`c2` 并非“比 `c1` 具有更强格律模板限制的对照组”。`c1` 是直接给出《沁园春》完整显式格律模板的一组；而 `c2` 虽然固定了《沁园春》与“改革”主题，但仅通过简短的文字（提醒注意词林正韵与钦定词谱）来约束模型。

### 1.4 Prompt 指令的具体内涵

为避免使用 `type_a` 至 `type_e` 的代号导致理解不够直观，此处对各指令的具体含义进行阐释：

- `type_c`：模型自主选择词牌、自主决定主题，自由创作一首宋词。
- `type_b`：词牌限定在给定范围内，但不规定创作主题。
- `type_d`：同时限定词牌和具体的主题背景。
- `type_a`：固定词牌为《沁园春》，并在提示词中直接提供较完整的格律骨架。
- `type_e`：固定词牌为《沁园春》，强调需符合词林正韵与钦定词谱规范，但不再展示完整的格律骨架。

设定中的两个主题也对应着具体的情境叙事，而非抽象的标签：

- 改革主题：假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。
- 玉兰主题：假设你是一个喜欢写宋词的大学生，最近在学业和面试上屡遇挫折，但是你看到了一颗盛开着白玉兰花的树，似乎重新找到了生命的实在与价值，便写了一首词。

## 二、AI 生成宋词的格律与结构准确性

### 2.1 准确性的具体评估维度

本节所探讨的“准确性”，侧重于文本的形式合规层面，而非文学造诣的高低。系统实际校验的维度涵盖以下四项：

1. 字数是否符合词牌规定。
2. 平仄规律是否基本成立。
3. 韵脚选用与押韵位置是否准确。
4. 在长调创作中，整体句式结构是否能保持连贯。

简而言之，本节评估的是“这首词是否具备基本的形式合理性”，而非探讨其“是否达到了经典名家的文学高度”。

### 2.2 格律评分的计算逻辑

`engine/songci_engine.py` 的计分逻辑明确：基准分为 `100` 分，每出现一种格律违规现象便按对应权重扣分，最终按以下公式计算：

```text
score = max(0, 100 - total_penalty × 4)
```

系统设定的惩罚梯度大致为：

- 平仄错误：按较高权重扣分。
- 韵脚声调错误：惩罚力度重于普通平仄错误。
- 出韵、挤韵：施加最高级别惩罚。
- 未收录字：仅作警告提示，不计入扣分。

此外，由于字数不符会直接导致该词谱变体匹配失败，因此大量“0分”样本并非源于“平仄略有瑕疵”，而是其整体字数与框架结构出现了根本性偏离。

### 2.3 任务条件对格律的影响：引入主题会导致格律表现下降

若不局限于数据库编号（a1/b1等），而是按具体的任务约束条件来观察，数据规律会更为清晰：

| 任务说明 | 对应代码 | 平均格律分 | 100 分比例 | 高质量比例 | 0 分比例 |
|---|---|---:|---:|---:|---:|
| 完全自由生成 | `a1` / `type_c` | 70.37 | 49.0% | 59.7% | 19.9% |
| 固定五个词牌，不给主题 | `b1` / `type_b` | 71.74 | 54.5% | 62.8% | 19.5% |
| 固定五个词牌，给改革主题 | `b2` / `type_d` | 63.03 | 40.1% | 50.6% | 24.0% |
| 固定五个词牌，给玉兰主题 | `b3` / `type_d` | 65.55 | 44.3% | 54.7% | 25.2% |
| 固定 `沁园春`，给玉兰主题，并附显式词谱模板 | `c1` / `type_a` | 61.79 | 41.6% | 53.5% | 31.4% |
| 固定 `沁园春`，给改革主题，只做简略格律提醒 | `c2` / `type_e` | 44.24 | 22.7% | 34.1% | 45.1% |

此处呈现出两个显著的客观事实：

第一，**主题一旦介入，格律得分便出现明显下降。**  
从“固定五个词牌、无主题”的 `71.74` 分，下降至“固定五个词牌 + 改革主题”的 `63.03` 分，以及“固定五个词牌 + 玉兰主题”的 `65.55` 分，降幅显著。这表明模型一旦需要同时兼顾具体情境、人物身份与叙事背景时，其对形式格律的控制力会被严重挤压。

第二，**提供格律提示并不意味着必然能生成完全合规的作品。**  
`c1` 任务虽已直接提供了《沁园春》的显式词谱模板，但平均格律分仅为 `61.79`，高质量比例为 `53.5%`，且 0 分比例仍高达 `31.4%`。这说明显式模板具备辅助作用，但绝非确保模型合规的充分条件。此结论必须在报告中予以明确。

### 2.4 提示词精细度对《沁园春》格律表现的影响

为了清晰评估“词谱模板”的实际效用，不能仅看整体均分，必须单独提取《沁园春》这一词牌进行对照。

下表将涉及《沁园春》的所有任务进行横向对比：

| `沁园春` 任务说明 | 对应代码 | 平均格律分 | 高质量比例 | 100 分比例 | 0 分比例 |
|---|---|---:|---:|---:|---:|
| 选定 `沁园春`，无主题、无模板 | `b1` 中提取 | 56.90 | 46.5% | 36.6% | 31.7% |
| 选定 `沁园春`，改革主题、无模板 | `b2` 中提取 | 56.93 | 43.0% | 29.9% | 28.0% |
| 选定 `沁园春`，玉兰主题、无模板 | `b3` 中提取 | 56.86 | 45.7% | 32.8% | 33.6% |
| 固定 `沁园春`，玉兰主题，显式词谱模板 | `c1` | 61.65 | 53.4% | 41.3% | 31.5% |
| 固定 `沁园春`，改革主题，简略格律提醒 | `c2` | 44.24 | 34.1% | 22.7% | 45.1% |

这一组对比揭示了三个重要现象：

1. **显式模板确实能提升表现，但幅度有限。**  
同样创作玉兰主题的《沁园春》，从“不提供模板”的 `56.86` 分，提升至“提供显式模板”的 `61.65` 分，仅有有限的改善，并未达到完全准确的标准。
2. **简略的文字提醒作用甚微。**  
同为改革主题，`b2` 中自然生成的《沁园春》均分尚有 `56.93`，而到了“仅提供简略格律提醒”的 `c2` 组，得分反而大幅跌至 `44.24`。这表明仅仅在提示词中强调“注意词林正韵、注意钦定词谱”无法替代显式模板的结构性支撑。
3. **模板的辅助效果因模型而异。**  
`c1` 的 0 分比例依然高达 `31.5%`，且不同模型在接收到模板后的表现收益差异巨大。

### 2.5 模型差异：不同模型对模板与主题的敏感度不同

三款主力模型在格律表现上呈现出显著的内部差异：

| 任务说明 | `doubao-seed-2.0-pro` | `deepseek-3.2-thinking` | `qwen3.6-plus` |
|---|---:|---:|---:|
| 完全自由生成 | 94.32 | 69.52 | 64.49 |
| 五词牌、无主题 | 92.12 | 68.80 | 61.98 |
| 五词牌、改革主题 | 81.08 | 51.92 | 59.43 |
| 五词牌、玉兰主题 | 87.04 | 55.02 | 53.59 |
| `沁园春` + 玉兰 + 显式模板 | 74.66 | 63.48 | 41.72 |
| `沁园春` + 改革 + 简略提醒 | 79.76 | 18.59 | 28.68 |

若仅观察总计数据，容易忽略一个关键事实：**模板的有效性高度依赖于模型本身的固有特性。**

以《沁园春》的生成为例：
- `deepseek` 在玉兰主题下，得分从无模板的 `33.94` 跃升至显式模板（`c1`）下的 `63.48`，表明该模型高度依赖外部提供的结构骨架。
- `qwen3.6-plus` 在玉兰主题下，得分从无模板的 `45.89` 降至有模板的 `41.72`，基本未从显式模板中获益。这说明“模型接收到模板指令”与“模型能够准确执行模板”是两个独立的环节。
- `doubao` 在五词牌玉兰主题下的《沁园春》得分为 `83.47`，进入 `c1` 后反而降至 `74.66`。这表明其自身结构维持能力已经较强，强行嵌入外部模板未必能带来正向叠加收益。

### 2.6 主要的格律错误类型

从错误类型看，最常见的不是押韵，而是平仄和长调结构持续性。

在数据库层面的统计中：
- `b1` 组中，“应平实仄”类错误覆盖了 `24.3%` 的作品，“应仄实平”覆盖了 `26.8%`。
- 在“改革”主题约束下，上述两项错误比例分别升至 `36.9%` 与 `31.7%`。
- 在提供显式模板的 `c1` 组中，比例进一步攀升至 `44.8%` 与 `35.9%`。
- 仅提供简略提醒的 `c2` 组最为严重，错误覆盖率分别高达 `60.2%` 与 `49.2%`。

这说明在长调与强主题双重约束下，模型最先丧失的往往不是局部押韵，而是整体句式和平仄交替的持续维持能力。

### 2.9 典型格律错误案例

仅靠比例表还不够，还需要看错误究竟长什么样。

#### 例 1：长调中连续平仄失配

数据来源：`c2 / deepseek-3.2-thinking / 沁园春 / 宗派感怀`

正文片段：
> 规锁灵台，法缚心猿，旧制如磐。叹弟子抑郁，青春虚度；门庭肃杀，灵气萧然……

引擎提示中连续出现：
- “规”平声，此处须仄声字
- “缚”去声，此处须平声字
- “猿”平声，此处须仄声字
- “子”上声，此处须平声字

这类样本表明，模型虽然还能组织完整语义，但在长调平仄链条上已经失去连续控制能力。

#### 例 2：严重字数不符

数据来源：`c2 / qwen3.6-plus / 沁园春 / 玄枢阁偶感`

正文片段：
> 铁律森森，云壑沉沉，剑阵列霜。叹青衿缚羽，空怀星斗；丹炉锁雾，暗减韶光……

引擎错误：
> 该变体要求 105 字，实际 56 字，相差 49 字。

这说明有些失败不是“局部小错”，而是根本没有真正生成完整的《沁园春》结构。

#### 例 3：只差 1 字也会被判 0 分

数据来源：`c2 / qwen3.6-plus / 玄渊阁规谏`

引擎错误：
> 该变体要求 105 字，实际 106 字，相差 1 字。

这提醒我们，词体格律评估在字数上采取的是硬约束判定。对现代文本生成来说，这看似苛刻；但对词体研究而言，它恰恰说明“字数控制”本身就是能力边界的一部分。

#### 例 4：词牌变体智能纠偏

在 `b1/b2/b3` 中常出现如下提示：

> 智能纠偏：该词字数/格律实际匹配的是「浪淘沙令」(54字)，而非「浪淘沙」。

这一问题出现频次并不低，例如：
- `b1 / deepseek`：`36` 次
- `b1 / doubao`：`36` 次
- `b2 / doubao`：`34` 次

这类情况不应简单归为“完全写坏”，它更像是模型在相近词牌变体之间发生了结构滑移，因此应与严重长调崩溃区分看待。

### 2.10 词牌难度的天然梯度

将全部样本合并按词牌分类后，可观察到显著的总体难度差异：

| 词牌 | 平均格律分 | 高质量比例 | 0 分比例 |
|---|---:|---:|---:|
| `清平乐` | 81.33 | 74.0% | 13.1% |
| `浪淘沙` | 78.18 | 73.4% | 16.5% |
| `菩萨蛮` | 67.07 | 49.5% | 16.5% |
| `沁园春` | 56.28 | 46.5% | 34.5% |
| `祝英台近` | 50.29 | 38.8% | 37.7% |

这一总体梯度与前面的 `b1` 受控排名、极端组合表和错误案例是相互印证的：`清平乐`、`浪淘沙` 更像模型可以稳定把握的常规区间，而 `沁园春`、`祝英台近` 则构成了项目中最典型的高风险结构测试。

### 2.11 本节结论

本节需重点强调以下六个结论：
1. 主题设定的引入会大幅降低模型的格律准确率，这是本实验中极为稳定的数据现象。
2. 显式词谱模板具有辅助作用，但无法保证完美的输出合规率。
3. 在受控条件下，`浪淘沙` 与 `清平乐` 明显更易生成，而 `沁园春` 与 `祝英台近` 是稳定的高难词牌。
4. 最危险的失败不是单个押韵问题，而是长调中的连续平仄失控与字数整体崩塌。
5. “词牌变体纠偏”是一类需要单独识别的问题，它不同于彻底的格律失控。
6. 格律表现高度受制于模型差异：`doubao` 形式控制最为稳定，`deepseek` 最依赖外部结构框架，而 `qwen` 在处理长调格式时最易出错。

## 三、AI 模型的意象偏好

### 3.1 `a1` 中模型自主选择词牌的差异

在完全自由生成的 `a1` 中，模型不仅自己决定写什么，还自己决定用什么词牌。这个环节本身就能反映模型的默认创作重心。

| 模型 | 作品数 | 词牌数 | Top1 词牌 | Top1 占比 | 熵 |
|---|---:|---:|---|---:|---:|
| `qwen3-max` | 48 | 2 | `青玉案` | 0.8125 | 0.6962 |
| `doubao-seed-2.0-pro` | 50 | 4 | `鹧鸪天` | 0.5200 | 1.4663 |
| `qwen3.6-plus` | 49 | 9 | `鹧鸪天` | 0.4286 | 2.4187 |
| `glm-5` | 50 | 18 | `鹧鸪天` | 0.2400 | 3.6962 |
| `glm-4.7` | 45 | 19 | `满江红` | 0.1333 | 3.9690 |

这张表很重要，因为它说明所谓“自由生成”并不意味着模型会均匀探索各种词牌。相反，不少模型会迅速回到自己最熟悉的少数词牌上。`qwen3-max` 几乎把 `a1` 写成了 `青玉案` 的单一通道；`doubao` 也明显偏向 `鹧鸪天`。只有 `glm-5` 与 `glm-4.7` 在词牌选择上表现出更高分散度。

### 3.2 无主题限制下的默认意象

首先考察两类弱约束任务：完全自由生成与固定五词牌但无主题。

在“完全自由生成”状态下，最高频的意象依次为：

| 意象 | 覆盖比例 |
|---|---:|
| 西风 | 16.7% |
| 孤舟 | 13.7% |
| 芦花 | 12.9% |
| 明月 | 11.7% |
| 星河 | 8.5% |
| 烟波 | 8.2% |

在“固定五词牌、无主题”状态下，高频意象转变为：

| 意象 | 覆盖比例 |
|---|---:|
| 明月 | 15.7% |
| 孤舟 | 12.7% |
| 西风 | 12.0% |
| 暮云 | 10.6% |
| 雁字 | 10.3% |
| 斜阳 | 8.9% |

这说明，弱约束并不等于真正发散。在缺乏具体内容指引时，模型会迅速回调到一套高度成熟的传统宋词默认语料，把西风、孤舟、雁字、斜阳、暮云等作为安全且高频的起手式。

### 3.3 词牌对意象分布的引导作用

词牌不仅是字数和平仄的形式框架，也会二次影响景物选择与情绪基调：

- `菩萨蛮`：多用明月、孤舟、芦花、西风、秋江，整体偏清婉。
- `清平乐`：多用明月、西风、斜阳、清风、流水，更轻灵。
- `浪淘沙`：多用孤舟、烟水、荻花、雁字、暮色，极易滑向羁旅江景。
- `祝英台近`：多用暮云、西风、明月、阑干、孤棹、晚烟，局部景物更细密。
- `沁园春`：多用明月、雁阵、斜阳、芦花、落日，更偏宏大远景。

因此，词牌约束不仅改变形式，也在悄悄改变模型“会写成什么样的景”。

### 3.4 `a1 / b1` 无主题条件下的模型级意象集中度

仅看数据库整体高频意象还不够，还需要看模型自身会不会在无主题条件下就滑向固定模板。这里使用 `top10_imagery_ratio` 观察模型对高频意象的集中度。

| 条件 | 模型 | `top10_imagery_ratio` | 最常出现的 4 个意象 |
|---|---|---:|---|
| `a1` | `qwen3-max` | 0.3088 | 芦花、西风、孤舟、烟水 |
| `a1` | `deepseek-3.2-thinking` | 0.1982 | 数据、代码、算法、星河 |
| `a1` | `qwen3.6-plus` | 0.1472 | 孤舟、雁字、西风、芦花 |
| `a1` | `doubao-seed-2.0-pro` | 0.0984 | 清露、岸柳、石径、晚风 |
| `b1` | `qwen3.6-plus` | 0.1363 | 孤舟、雁字、芦花、暮云 |
| `b1` | `deepseek-3.2-thinking` | 0.1098 | 明月、暮云、数据、星河 |
| `b1` | `doubao-seed-2.0-pro` | 0.0794 | 西风、疏钟、汀洲、明月 |

这组数据说明三件事。第一，`qwen3-max` 在 `a1` 中已经显示出非常强的自发模板化倾向。第二，`doubao` 的默认意象分布最分散，更像是在传统意象池中做较宽松的调度。第三，`deepseek` 即使进入 `b1`，仍会残留 `数据`、`代码`、`算法` 这一类现代语汇，说明其“现代语义泄漏”并不是偶发噪音，而是一种稳定特征。

### 3.5 不同模型的默认意象特征

三款主力模型在默认状态下表现出明显不同的意象底色。

#### `doubao`
`doubao` 倾向于园亭山水与清疏小景。完全自由生成时常调用 `清露`、`岸柳`、`石径`、`晚风`；进入 `b1` 后，则偏向 `西风`、`疏钟`、`汀洲`、`明月`。整体古典感最稳定，现代词残留极少。

#### `deepseek`
`deepseek` 的特征最突出。在 `a1` 中它频繁输出 `数据`、`代码`、`算法`、`网络` 等带现代印记的概念词；即使在 `b1` 中整体风格被拉回古典语境，这些词仍出现在高频表中。它并非不会写传统景物，而是容易把现代概念和古典抒情框架混写在一起。

#### `qwen3.6-plus`
`qwen3.6-plus` 的默认意象高度集中于“离愁羁旅”一类语料。无论是否限定词牌，它都会反复调用 `孤舟`、`雁字`、`芦花`、`暮云`、`长亭` 这一套意象，因此古典化程度较高，但也最容易落入固定的“秋江送别”模板。

### 3.7 改革主题对意象分布的影响

在改革主题提示下，高频意象迅速收束为“宗门制度、修行秩序、内心挣扎”一组相互关联的词汇。
改革主题原文设定为：
> 假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。

在 `b2` 中，高频意象主要是：
- 樊笼
- 劫火
- 灵根
- 剑气
- 铁律
- 青衿
- 星斗

在 `c2` 中，进一步集中为：
- 青衿
- 灵根
- 樊笼
- 丹炉
- 晨钟
- 暮鼓
- 仙峰
- 弟子

这说明模型并没有把“改革”发散为广义社会议题，而是非常稳定地把它翻译成一套“宗门旧规、修炼秩序、危机压力、青年弟子”的修仙词汇系统。

### 3.8 玉兰主题对意象分布的影响

玉兰主题原文设定为：
> 假设你是一个喜欢写宋词的大学生，最近在学业和面试上屡遇挫折，但是你看到了一颗盛开着白玉兰花的树，似乎重新找到了生命的实在与价值，便写了一首词。

玉兰主题的收束更单一，也更稳定。

在 `b3` 中，高频意象为：
- 玉兰
- 寒窗
- 素萼
- 玉树
- 暗香
- 春风

在 `c1` 中，则进一步收缩为：
- 玉兰
- 琼枝
- 寒窗
- 琼英
- 玉树
- 清芬

也就是说，玉兰主题几乎天然地把模型导向一条固定路径：先写花之洁白清香，再写寒窗受挫，最后落脚到借花振作的情感转折。

### 3.9 `category -> imagery`：题材类型如何对应固定意象

数据库级高频意象还不够细，还需要看不同题材类型内部偏好哪些意象，以及哪些意象是某一类题材相对于全库更偏好的“标志性意象”。

### 3.11 本节结论

本节的核心发现可归纳为六点：
1. 在 `a1` 中，模型对词牌的自主选择已经表现出明显偏好，自由生成并不是真正的均匀探索。
2. 在弱约束条件下，AI 会自动回调到一套高度成熟的传统宋词默认意象库。
3. 词牌会对景物选择和情绪基调产生二次过滤作用。
4. 不同模型拥有不同的默认意象底色，其中 `doubao` 更分散、`qwen` 更容易滑向羁旅模板、`deepseek` 更容易混入现代语义。
5. 不同题材类型内部已经形成较稳定的 `category -> imagery` 对应关系，说明 AI 的“自由写作”内部仍有明显的子模板结构。
6. 同一主题在不同数据库之间共享较高比例的核心意象池，这为后文讨论主题收束与同质化奠定了直接证据。

## 四、AI 实际生成的内容题材

### 4.1 题材分类框架

本项目中的“主题分类”并非完全开放式的文本聚类生成，而是将生成作品归拢至预设的传统词题范畴中。在弱主题或无主题任务中，作品主要落在以下九大类别的候选框内：
1. 离别羁旅
2. 闲适隐逸
3. 豪情壮志
4. 悼亡相思
5. 咏物寄托
6. 谈禅说理
7. 怀古咏史
8. 山水田园
9. 春怨秋悲

因此，探讨题材分布时，需明确其本质是评估模型在上述候选类型中的归类滑移倾向。

### 4.2 各数据库的 Top1 / Top2 题材汇总

在进入分库细读之前，先用一张汇总表把六个数据库最主要的题材归类压缩出来：

| 数据库 | Top1 类别 | 占比 | Top2 类别 | 占比 |
|---|---|---:|---|---:|
| `a1` | 离别羁旅 | 34.33% | 闲适隐逸 | 14.68% |
| `b1` | 离别羁旅 | 42.71% | 闲适隐逸 | 15.74% |
| `b2` | 谈禅说理 | 41.08% | 咏物寄托 | 39.65% |
| `b3` | 咏物寄托 | 92.05% | 豪情壮志 | 5.25% |
| `c1` | 咏物寄托 | 87.66% | 豪情壮志 | 10.89% |
| `c2` | 谈禅说理 | 57.20% | 豪情壮志 | 32.58% |

这张表把全章的结构关系一下子拉清了：无主题条件下，模型主要滑向 `离别羁旅`；改革主题下，模型主要在 `谈禅说理 / 咏物寄托 / 豪情壮志` 三类之间压缩；玉兰主题下，则几乎直接收束为 `咏物寄托`。

### 4.3 无主题约束并不等同于发散创作

观察 `a1` 与 `b1` 两组数据，可以看到即便没有给出主题，题材分布也不会真正发散。

完全自由生成时，占比最高的前五类为：
- 离别羁旅：`34.3%`
- 闲适隐逸：`14.7%`
- 豪情壮志：`10.2%`
- 悼亡相思：`9.2%`
- 咏物寄托：`9.0%`

添加词牌约束但不给主题后，题材收束进一步增强：
- 离别羁旅：`42.7%`
- 闲适隐逸：`15.7%`
- 怀古咏史：`9.8%`
- 悼亡相思：`7.6%`
- 豪情壮志：`7.1%`

也就是说，无主题并不意味着自由漫游，而更像是模型回落到自己最熟悉、最容易用传统意象堆砌完成的少数题材之中，其中尤以 `离别羁旅` 最为稳定。

### 4.4 不同模型在无主题环境下的默认题材偏向

即便在同样的无主题条件下，不同模型也有不同的默认题材重心。

在 `a1` 中：
- `doubao` 更偏 `闲适隐逸`、`山水田园` 与 `离别羁旅`。
- `deepseek` 更常落在 `咏物寄托`、`悼亡相思`、`豪情壮志` 与 `谈禅说理`。
- `qwen3.6-plus` 的 `离别羁旅` 占比高达 `59.2%`，集中度非常高。

在 `b1` 中：
- `doubao`：`离别羁旅` 占 `48.0%`，同时仍保留较高比例的 `闲适隐逸` 与 `怀古咏史`。
- `deepseek`：`离别羁旅` 仅占 `28.0%`，`豪情壮志` 与 `谈禅说理` 明显更高。
- `qwen3.6-plus`：`离别羁旅` 占 `57.4%`，是三款主模型里偏好最集中的代表。

这一层模型差异与前一章的意象画像是对应的：`qwen` 默认就容易写成羁旅抒情，`deepseek` 更容易进入概念化、议论化或带现代感的路径，`doubao` 则分布相对更宽。

### 4.5 改革主题的题材收束倾向

改革主题原文设定如下：
> 假设你是一个修仙世界一个宗派的年轻长老，看到了宗派中传统的规矩和观念对宗派中的弟子的压抑，希望能改变这个现实，但是发现严格的规矩的确能一定程度上能保证在危机面前的战斗力，而且宗派中很多长老也很反对这个改革。你在挣扎中写了一首词来抒发自己的想法。

在这一设定下，模型并未把主题扩展成复杂的社会叙事，而是主要压缩为三种古典范式：

- `b2`：谈禅说理 `41.1%`，咏物寄托 `39.6%`，豪情壮志 `19.3%`
- `c2`：谈禅说理 `57.2%`，豪情壮志 `32.6%`，咏物寄托 `9.8%`

如果进一步拆到模型层，差异会更清楚：

| 模型 | `b2` 中最主要的题材归类 |
|---|---|
| `doubao-seed-2.0-pro` | `谈禅说理` `63.00%` |
| `qwen3.6-plus` | `咏物寄托` `41.61%`，`豪情壮志` `31.68%` |
| `deepseek-3.2-thinking` | `咏物寄托` `39.00%`，`谈禅说理` `38.50%` |

因此，改革主题并不是被所有模型“同一种方式”翻译的。`doubao` 更倾向把它写成带制度反思色彩的议论词，`deepseek` 在议论与借物寄托之间摇摆，`qwen` 则更容易把其推向情绪更外放的豪情框架。

### 4.6 玉兰主题的题材收束倾向

玉兰主题原文同样需要保留：
> 假设你是一个喜欢写宋词的大学生，最近在学业和面试上屡遇挫折，但是你看到了一颗盛开着白玉兰花的树，似乎重新找到了生命的实在与价值，便写了一首词。

在这一设定下，模型的分类归拢呈现出极高的一致性，几乎稳定转化为 `咏物寄托`：

- `b3`：`咏物寄托` 占 `92.05%`
- `c1`：`咏物寄托` 占 `87.66%`

模型之间虽然有细微差异，但都没有改变总方向：
- `doubao`：`91.5%` 或 `84.0%`
- `deepseek`：在 `c1` 中高达 `96.5%`
- `qwen3.6-plus`：分别达到 `91.5%` 与 `81.2%`

这说明玉兰主题是本项目中最强的题材收束场景之一。无论模型还是形式约束如何变化，它们几乎都会把这一叙事写成“借花写人、借花自振”的传统咏物路径。

### 4.7 本节结论

1. AI 生成的题材归类并非无限开放，而是在既定古典题材候选池中游移。
2. 无主题条件下，模型会自动回到少数熟悉题材，尤其是 `离别羁旅`。
3. 数据库级 Top1 / Top2 汇总表清楚表明，主题变化首先改变的是题材分布结构，而不是局部词句。
4. 改革主题主要被压缩为 `谈禅说理 / 咏物寄托 / 豪情壮志` 三条路径，但不同模型的落点并不一样。
5. 玉兰主题具有最强的题材收束效应，几乎稳定指向 `咏物寄托`。

## 五、AI 生成文本与真实宋词的相似性分析

### 5.1 分析逻辑：建立历史基线与评估借用关系

在探讨文本相似性时，为确保严谨性，不宜简单使用“抄袭”等定性词汇。本研究遵循以下三步逻辑进行论证：
1. 首先量化真实历史宋词体系内部固有的相似度分布（建立历史基线）。
2. 其次评估 AI 整体作品与某一首真实宋词文本的接近程度。
3. 最后考察 AI 作品中的局部语句是否高频度地追溯复刻同一首真实来源。

### 5.2 数据规模与历史基线特征

本阶段的相似性测算涵盖了庞大的数据基数：
- AI 作品 `3122` 首
- AI 句子 `49304` 句
- 真实宋词 `21053` 首
- 真实句子 `270107` 句

经过对比测算，真实历史宋词内部本身就存在较高的相互借鉴度。其加权汇总的基线数据如下：
- 整首语义相似度均值约为 `0.8404`
- 整首字面相似度均值约为 `0.2366`
- 句级语义相似度均值约为 `0.7318`
- 句级字面相似度均值约为 `0.4415`

更为关键的是，在 AI 与真实语料的比对中，共检出 `2145` 条句级字面相似度达 `1.0` 的完全匹配项，但对其结构拆解后发现：
- `2127` 条隶属于 `<=5` 字的短句。
- `18` 条隶属于 `6-8` 字的句子。
- 9 字以上的长句几乎不存在完全一致的匹配。

这证明，绝对的字面一致主要集中在短句与高频的格式化套语层面，不可将此类重叠一概定性为异常的文本剽窃。

### 5.3 整体相似性评估结论

从整首作品维度的匹配结果来看：
- 整首语义分位达到历史库 `P95` 及以上的作品为 `45` 首，占比 `1.44%`
- 整首字面分位达到历史库 `P95` 及以上的作品为 `46` 首，占比 `1.47%`
- 语义主来源覆盖率 `>= 0.2` 的作品为 `30` 首，占比 `0.96%`
- 字面主来源覆盖率 `>= 0.2` 的作品为 `41` 首，占比 `1.31%`
- 主来源覆盖率 `>= 0.5`（即半数以上句子来源于同一古词）的作品数量，在语义与字面维度均为 `0`。

数据表明，在所有生成的 `3122` 首作品中，几乎不存在基于单一历史来源全篇重写的现象。绝大多数样本的生成模式，更接近于从庞大的传统宋词意象池中，分散抽调意境元素与成熟的抒情构件进行重组。

### 5.4 任务约束与相似性的交互关系

不同任务条件下的相似度特征统计如下：

| 任务说明 | 平均整首语义分 | 平均语义分位 | 平均整首字面分 | 平均字面分位 |
|---|---:|---:|---:|---:|
| 完全自由生成 | 0.8539 | 0.6916 | 0.1991 | 0.3376 |
| 五词牌、无主题 | 0.8524 | 0.6326 | 0.2099 | 0.3640 |
| 五词牌、改革主题 | 0.8352 | 0.5040 | 0.1880 | 0.1476 |
| 五词牌、玉兰主题 | 0.8477 | 0.6032 | 0.1938 | 0.2004 |
| 固定 `沁园春`、玉兰主题、显式模板 | 0.8723 | 0.2453 | 0.1792 | 0.1694 |
| 固定 `沁园春`、改革主题、简略提醒 | 0.8695 | 0.2053 | 0.1788 | 0.1289 |

此类分布揭示出：
1. `c1` 与 `c2` 任务的绝对整首语义得分很高，但其历史分位值却相对较低。这说明固定《沁园春》这一长调格式确实容易生成极具传统质感的文本，但这主要归因于《沁园春》自身在历史文献库中的基线原本就较高，而非发生了异常的集中重合。
2. 真正容易触及历史相似度高尾部的，反而是自由或半自由的无约束条件。因为在这些任务中，模型更容易直接滑入成熟且惯用的词境模板。

### 5.5 不同词牌的相似性风险特征

在“五词牌、不规定主题”的对照组中，各词牌展现出不同的风险形态：
- `祝英台近`：句级语义 `P95` 比例为 `56.2%`，句级字面 `P95` 比例高达 `70.1%`，是局部借用与改写风险最高的词牌类型。
- `浪淘沙`：整首语义分位均值达 `0.7627`，呈现出整体性坠入传统意境套用的特征。
- `沁园春`：其整首字面最近邻追溯至同名历史词牌的比例为 `32.4%`，同词牌吸附效应最为明显，但该现象必须结合其自身极高的历史基线进行审慎解读。
- `清平乐` 与 `菩萨蛮`：表现更接近常规的模块化拼装，未表现出强烈的局部借用敏感性。

### 5.6 相似性文本溯源的具体案例

为了清晰阐释“借用关系”的具体表征，此处必须引入典型的实证案例。

#### 例一：高语义贴近，但来源分散，非整首重写
样本：`a1:81`，`qwen3-max`，`青玉案《秋江夜泊》`
AI 文本片段：
> 西风卷尽芦花雪，又送客、孤舟发。月照寒汀烟水阔。

分析：该词整首语义分位高达 `0.9781`，最近邻整体指向方岳的《齐天乐》，但主来源覆盖率仅为 `0.0833`。其实质并非在重写某一首词，而是将传统羁旅词中极为成熟的一组意象构件进行了高密度的重新拼装。

#### 例二：局部句法骨架向单一来源高度收敛
样本：`b2:364`，`deepseek-3.2-thinking`，`祝英台近《仙门感怀》`
AI 文本片段：
> 怕上层楼，危机暗中满。
> 断肠规矩如链，青春空晚，更谁劝、莺声留挽？

分析：该词整首字面分位达到顶峰 `1.0000`，字面主来源覆盖率达 `0.25`，最近邻溯源至辛弃疾的《祝英台令》。其局部字词对应关系极为直接：
- `怕上层楼` 对 `怕上层楼`
- `更谁劝、莺声留挽` 对 `倩谁唤、流莺声住`
此类情况已超出常规的“传统共相”，呈现出明显的局部改写式借用特征。

#### 例三：长调强约束下向名作的严重贴合
样本：`c2:193`，`deepseek-3.2-thinking`，`沁园春《宗规感怀》`
AI 文本片段：
> 世路艰险，劳生有限，似此压抑长难容。微吟罢，凭栏杆无语，心事重重。
> 有宏图千卷，胸中万策；致君尧舜，此事非空。
> 用舍由时，行藏在我……

分析：整首字面分位达 `0.9905`，语义和字面的主来源覆盖率分别高达 `0.36` 和 `0.32`，最近邻显著指向苏轼的名作《沁园春》。其文本重合关系极度贴近：
- `劳生有限` 对 `劳生有限`
- `微吟罢` 对 `微吟罢`
- `胸中万策` 对 `胸中万卷`
- `致君尧舜` 对 `致君尧舜`
- `用舍由时，行藏在我` 对 `用舍由时，行藏在我`
此类案例必须引起警惕并进行人工复核，因其不仅在整体上模仿了某类词风，更是在关键结构节点上持续倒向特定的历史名作。

### 5.7 相似性风险的模型分布差异

若设定高风险阈值为：
- 整首字面分位进入历史 `P95`
- 且字面主来源覆盖率 `>= 0.2`

全数据库符合该条件的仅有 `15` 首作品，且**这 `15` 首全部出自 `deepseek-3.2-thinking` 模型**。

其任务分布主要集中于：
- 固定 `沁园春` + 改革主题 + 简略提醒：`6` 首
- 五词牌 + 改革主题：`5` 首
- 五词牌、不规定主题：`4` 首

其词牌分布主要集中于：
- `沁园春`：`11` 首
- `祝英台近`：`4` 首

结论非常明确：文本借用风险并非在整个生成体系中均匀分布，而是高度依赖于特定模型自身的特性，并受特定任务参数叠加放大的影响。

### 5.8 本节结论

1. 绝大部分 AI 宋词并未表现出“基于单一历史来源全篇重写”的特征。
2. 模型的创作常态是对庞大历史意象池、短句套语及成熟词境的高密度复用。
3. 真正值得警惕的是局部文本过度向单一来源聚拢的现象，尤以 `deepseek` 在处理《祝英台近》与《沁园春》时的部分样本最为典型。
4. 基于上述实证，将此类现象表述为“借鉴”、“借用关系”或“来源相似性”更为客观严谨，不宜不加区分地统称为“抄袭”。

## 六、AI 生成文本的内部同质化分析

### 6.1 核心概念辨析：字面相似与语义相似

在评估同质化指标前，必须通过具体案例界定两种相似度的本质区别，避免由于概念抽象而产生理解偏差。

**语义相似（换词不换意）：**
- 例 1：AI 文本：`素萼破寒轻绽`，真实来源：`绛萼欺寒`（语义相似度约 `0.9001`，字面相似度 `0.4`。字面用词差异大，但核心表达结构高度一致。）
- 例 2：AI 文本：`几度风萍浪梗`，真实来源：`此际浪萍风梗`（语义相似度约 `0.9045`，字面相似度 `0.4444`。词序与副词已变，但表达的意境指向完全一致。）

**字面相似（高度重合）：**

- 例 3：AI 文本：`银汉无声转玉盘`，真实来源：`银汗无声转玉盘`（字面相似度约 `0.8571`，语义相似度约 `0.9003`。仅一字之差，属于极强的局部文本贴近。）
- 例 4：AI 文本：`暗香浮动黄昏后`，真实来源：`暗香浮动黄昏后`（字面与语义完全一致的绝对重合案例。）

综上，两者不可混为一谈：语义相似偏向考察“是否沿用同一套表达骨架”，而字面相似则用于判定“具体语句是否陷入了机械复刻”。

### 6.2 关键指标

为便于后续论述，本节将程序化字段统一转译为规范的中文术语：
- `rhythm_score`：格律分
- `whole_score`：整词同质化度
- `sent_score`：句子同质化度
- `imagery_score`：意象同质化度
- `core_mihi`：核心同质化度
- `mihi_full`：总同质化度
- `excess_core_homogeneity`：超额核心同质化度

### 6.3 同质化评估的核心逻辑：区分整词收束与局部模板化

这是本节评估同质化现象时最关键的认知基础。

同题写作本来就应该在整首层面更接近。  
如果几百首“改革主题”或“玉兰主题”的词，在整体情境、主导情绪、意象方向上完全不接近，那反而说明 prompt 没有起作用。

因此，**整词同质化度上升，在主题约束下是正常现象，也是应该发生的现象。**  
真正需要警惕的是：句子同质化度是不是也被一并推高，导致局部表达开始模板化、套语化。

### 6.4 用“整词同质化度 - 句子同质化度”的差值来理解这个问题

通过计算“整词同质化度 - 句子同质化度”的差值，可以直观地辨别模型是正常的主题相同还是局部句子重复。

基准判定依据如下：
- 差值 `<= 0.18`：句子层级的数据紧迫逼近整词层级，代表局部模板化风险较高。
- 差值在 `0.18 - 0.30`：主要体现为宏观整词层面的收束，局部模板化风险处于中等水平。
- 差值 `>= 0.30`：宏观整词收束极为强烈，显著高于局部句子的趋同程度，此时文本展现出“同题写作总体相似，但局部句法尚有一定差异”的形态。

各任务表现均值对照：

| 任务说明 | 整词同质化度均值 | 句子同质化度均值 | 二者差值 | 用差值做的快速判断 |
|---|---:|---:|---:|---|
| 五词牌、无主题 | 0.6501 | 0.4953 | 0.1548 | 句子层风险偏重 |
| 五词牌、改革主题 | 0.7263 | 0.4384 | 0.2880 | 中等，主要是整词收束 |
| 五词牌、玉兰主题 | 0.7104 | 0.4728 | 0.2377 | 中等，主要是整词收束 |
| 固定 `沁园春`、玉兰主题、显式模板 | 0.8951 | 0.5362 | 0.3589 | 整词收束很强，句子层相对次之 |
| 固定 `沁园春`、改革主题、简略提醒 | 0.9256 | 0.5082 | 0.4174 | 整词收束极强，句子层相对次之 |

这张表非常重要，因为它说明：

1. 一旦给定主题，整词同质化度会明显上升。
2. 但在“五词牌 + 主题”这一步，差值反而变大了。
3. 这说明主题首先影响的是“整首词写什么”，而不是“每一句都写得越来越像”。

换句话说，**主题 prompt 先带来的是合理的整词收束，而不是坏的句子模板化。**

若进一步下看到模型维度，分析同主题下由五词牌转向长调强约束时，差值产生的动态变化，情况将更加清晰：

**改革主题内部：`b2 -> c2` 的模型差值**

| 模型 | `b2` 差值 | `c2` 差值 | 差值变化 | 用差值做的快速判断 | 解释 |
|---|---:|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `0.2711` | `0.3579` | `+0.0868` | 从中等转向整词收束更强 | 整词与句子都不低，但进入 `c2` 后整词抬得更快，说明它首先在整体结构和主题框架上进一步收紧 |
| `deepseek-3.2-thinking` | `0.2800` | `0.4264` | `+0.1464` | 从中等转向整词收束极强 | 这里的 `c2` 取观察分析低样本值；差值变大说明 surviving 样本更多集中在整词层，但也意味着高格律可用样本太少 |
| `qwen3.6-plus` | `0.3128` | `0.4679` | `+0.1551` | 本来就偏整词收束，进入 `c2` 后更明显 | 句子层没有同步追高，更多是整词层继续收紧，而不是局部模板突然恶化 |

*注：`deepseek` 与 `qwen` 在 `c2` 任务中因符合基本格律要求的样本稀少，此处取观察分析值。*

**玉兰主题内部：`b3 -> c1` 的模型差值**

| 模型 | `b3` 差值 | `c1` 差值 | 差值变化 | 用差值做的快速判断 | 解释 |
|---|---:|---:|---:|---|---|
| `doubao-seed-2.0-pro` | `0.2450` | `0.3552` | `+0.1102` | 从中等转向整词收束更强 | `c1` 中句子层也高，但整词层抬得更快，说明显式模板先把整体结构做得更整齐 |
| `deepseek-3.2-thinking` | `0.2294` | `0.3547` | `+0.1253` | 从中等转向整词收束更强 | 进入 `c1` 后，整词层被显著拉紧，句子层也升，但没有追得和整词一样快 |
| `qwen3.6-plus` | `0.2386` | `0.3666` | `+0.1280` | 从中等转向整词收束更强 | 它在 `b3` 时已经有稳定主题收束，到了 `c1` 后更多是整词进一步靠近，而不是句子层突然爆发 |

把这两张表和前面的总体表放在一起，就能得到一个更清楚的判断：**同样是更强约束，不同模型虽然都会把差值拉大，但“拉大”的含义并不一样。有的模型是在整词层继续收紧，有的模型则是根本写不出像样的词。**

### 6.5 判断严重性时，不能只看差值变化，还要看差值本身落在什么区间

现在可以把这一部分的逻辑统一成三步：

1. 先看差值本身：判断当前更像“整词收束”还是“句子模板化”。
2. 再看差值变化：判断强约束之后压力是在向整词层移动，还是向句子层移动。
3. 最后再看句子同质化度本身：判断局部句法是否已经高到值得警惕。

因此，真正的风险并不是“只要同质化升高就危险”，而是下面两种情况：

- 差值本身偏小，说明句子层已经和整词层靠得很近。
- 或者差值虽然不小，但句子同质化度本身仍然很高，说明模型在整体很像的同时，局部表达也已经明显收紧。
从当前数据看：

- `b1` 的平均差值只有 `0.1548`，这反而说明无主题条件下，模型更容易直接滑向熟悉句法和套语。
- `b2` 与 `b3` 的差值扩大到 `0.2880` 和 `0.2377`，说明主题先把压力集中到整词层，这是相对正常的。
- `c1` 与 `c2` 的差值进一步扩大到 `0.3589` 与 `0.4174`，说明强约束后整词收束极强；但与此同时，`c1` 的句子同质化度均值仍高达 `0.5362`，`c2` 也达到 `0.5082`，所以不能简单把“大差值”理解成“完全不危险”。

更准确地说：

- 小差值：更像“句子层已经危险地跟上来了”。
- 大差值：更像“整体写得太像了”，但还要继续检查句子分是不是也已经过高。

### 6.6 模型对同质化风险层级的影响

真正决定风险如何演化与落地的，往往是模型固有的属性特征，而非任务名称的表面设定。

#### `doubao-seed-2.0-pro`
**表现特征**：格律控制最佳，整词结构维持能力最强。
从差值看，其在 `b1` 时差值仅 `0.1395`，句子层风险已然显现；至 `b2`、`b3` 时差值分别拉伸至 `0.2711`、`0.2450`，表明主题成功将其引向整词收束；进入强约束的 `c1`、`c2` 后，差值进一步升至 `0.3552` 与 `0.3579`。
这表明其很好的规范执行力，反而使其“整体极度工整稳健、整体较为雷同”。

#### `deepseek-3.2-thinking`
**表现特征**：受主题指令牵引最为剧烈，且外部来源借用风险最高。
其差值从 `b1` 的 `0.1416` 扩大至 `b2` 的 `0.2800` 与 `b3` 的 `0.2294`，表明对主题设定的响应极其敏锐，迅速从局部模板化转向整词收敛；进入 `c1` 与 `c2`（观察样本）后，差值升至 `0.3547` 与 `0.4264`。但结合前文可知，历史高度重合的尾部高风险样本几乎全集中于该模型身上。
其综合特征为：对主题极为顺从、意象高度收敛，潜在问题较多。

#### `qwen3.6-plus`
**表现特征**：自带强烈的题材偏好惯性，在长调处理上极易失控。
其在无约束的 `b1` 阶段差值已达 `0.1833`，表现出先天的题材集中倾向；进入 `b2`、`b3` 后差值为 `0.3128`、`0.2386`；在遭遇长调压力的 `c1` 与 `c2`（观察样本）中，差值升至 `0.3666` 与 `0.4679`。
这说明其同质化的演进并非在于句式的层层固化，而是源自长调带来的形式大面积崩盘，导致最终仅存活少量高度同质化的合格样本。

### 6.7 本节结论

本节需确立以下六个核心判断：

1. 评估同质化必须进行分层拆解，严格区分整词层面的宏观相似与句子层面的微观字面复刻，以免将同题写作带来的合理趋同误判为机械模板化。
2. “差值分析法”（整词 - 句子同质化）是前置研判的核心手段。差值过小预警句子局部已面临模板化危机；差值庞大则说明约束条件有效压制了主题的发散性。
3. 判断同质化时，第二步要再看差值变化。`b1 -> b2` 与 `b1 -> b3` 中差值扩大，说明主题先压缩的是整首词的内容方向；`b2 -> c2` 与 `b3 -> c1` 中差值继续扩大，则说明强约束进一步强化了整体收束
4. 主题任务下出现整词同质化度的抬升是必然且合理的现象，代表模型在积极响应指令。真正的危险信号是局部句子同质化指标脱离合理区间随之飙升，或在高位差值的掩护下已触及绝对的高限。
5. 三款主力模型具备截然不同的风险演化路径：`doubao` 倾向于生成高度工整的制式作品，可能存在部分句式重复的问题；`deepseek` 容易落入宏观主题收缩与微观局部复刻的双重困境，问题比较严重；而 `qwen` 极易在处理长调时失控从而仅留下少量极度相似的遗留样本，基本认为没有创作能力。
6. 综上所述，同质化评估不能粗暴归结为“某项任务是否更具同质化”，而是必须通过“差值基数 + 差值走势 + 模型固有特征”三维交叉验证，方可精准剥离出合理的指令响应、应当警惕的局部模板化，以及诱发不同风险的深层动因。

## 七、针对 AI 宋词生成的实践建议

### 7.1 如果目标是“尽量写对”

- 应当在提示词中明确指定目标词牌名，切忌让模型完全发散式生成。
- 在涉及《沁园春》等字数众多、结构复杂的长调时，必须提供详尽的显式格律模板支撑，仅凭借“注重格律”的简略文字指令极易导致模型失控。
- 生成后接入程序化的自动化校验模块。
- 使用较为先进的大模型，并开启思考模式。

### 7.2 如果目标是“尽量多样，不要都写成一类”

- 不要把词牌、主题、长调、强风格提示一次性全加上。
- 可以只固定词牌、放开主题，或只固定主题、放开词牌。
- 批量生成后再筛选，比一次性追求“单首直接完美”更容易保留变化空间。

### 7.3 如果目标是“减少模板化”

- 大模型产生模板化基本是必然的，如果想降低模板化，可以使用比较先进的模型，如`doubao-seed-2.0-pro`、`gemini-3.1-pro`或`claude-opus-4.6`等，并开启思考模式。
- 使用更详细的主题背景可以将同质中心脱离，在表现上可以减少模板化

### 7.4 如果目标是“降低与真实宋词过近的风险”

- 针对《祝英台近》与《沁园春》等比较困难的词牌保持警惕。
- 使用数据库进行相似度检查。
- 尽量使用较为先进的模型，如`doubao-seed-2.0-pro`、`gemini-3.1-pro`或`claude-opus-4.6`等，并开启思考模式。

### 7.5 最终总结

综合来看，大模型已经具备了一定的宋词生成能力，但这种能力高度不均衡：它们能在不少场合写出形式基本成立、意境也像样的作品，却也会在长调、强主题和批量生成中迅速暴露出平仄错误、题材收缩、意象固化和局部借用的风险；因此，真正可用的工作流不应是“把 AI 当成会写词的人”，而应是“把 AI 当成一个需要在明确约束、自动校验和人工复核中使用的写作引擎”。
