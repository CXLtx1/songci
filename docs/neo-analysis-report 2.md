# 大模型宋词生成项目综合分析报告（丰富版）

## 1. 项目目标与报告定位

本项目不是单一的“同质化分析”项目，而是一个围绕**大模型宋词生成能力**展开的综合研究项目。  
它同时关注三个彼此相关、但又相对独立的分析方向：

1. **格律能力**
模型能否在不同 prompt 和不同任务约束下生成格律上较为合格的宋词。

2. **内容分布与表达特征**
模型在词牌选择、主题类型和意象系统上呈现出怎样的偏好、收缩与差异。

3. **内部同质化**
同一模型在相同条件下生成的作品，是否在整体语义、句级表达和意象层面趋于重复。

因此，本报告的整体目标是：

**从格律、主题、意象和相似性四个层面，系统分析不同大模型的宋词生成能力与生成特征。**

## 3. Prompt 设计与实验变量

Prompt 设计是整个项目实验控制的核心，不同 prompt 不是简单换措辞，而是在控制不同变量。

### 3.1 `type_c`

对应 `a1`。

- 模型自由选择词牌
- 不给定主题
- 要求完整、合格、尽量多样

它主要观察模型在弱约束下的**默认词牌偏好**、**内容偏好**和**自发收缩路径**。

### 3.2 `type_b`

对应 `b1`。

- 固定词牌
- 不固定主题

它主要用于测量：**词牌约束本身是否提高作品的一致性，是否影响格律表现。**

### 3.3 `type_d`

对应 `b2`、`b3`。

- 固定词牌
- 固定主题和背景
- 明确鼓励创造力、文学性和避免抄袭

它主要用于测量：**即使 prompt 明确要求创造力，主题是否仍会压缩表达空间。**

### 3.4 `type_a`

对应 `c1`。

- 固定 `沁园春`
- 固定主题
- 提供显式格律模板

它是一种“**主题约束 + 结构约束 + 显式格律提示**”同时存在的强约束 prompt。

### 3.5 `type_e`

对应 `c2`。

- 固定 `沁园春`
- 固定主题
- 强调用韵和格律
- 没有 `type_d/type_a` 中“发挥创造力”“体现文学之美”“不要抄袭”的那段引导

它是一种更短、更硬、更结果导向的强约束 prompt。

## 4. 项目总体流程

项目流程可以概括为：

`Prompt 设计 -> 批量生成 -> 数据入库 -> 格律检查 -> 意象/主题提取 -> 格律统计 -> 描述性统计 -> 同质化分析 -> 结果整合`

对应模块包括：

- 多库生成实验
- `songci_engine` 格律检查
- 主题类型 `category` 提取
- 意象 `imagery` 提取
- 格律统计脚本
- 描述性统计脚本
- 同质化分析脚本

因此，本项目本质上是一套“生成 + 检查 + 统计 + 解释”的完整分析管线，而不是单一分析脚本。

### 5.6 典型格律错误例子

#### 例 1：长调中大量平仄失配

数据库：`c2`  
模型：`deepseek-3.2-thinking`  
词牌：`沁园春`  
题目：`宗派感怀`  
分数：`0`

正文片段：

> 规锁灵台，法缚心猿，旧制如磐。叹弟子抑郁，青春虚度；门庭肃杀，灵气萧然……

错误中连续出现：

- `「规」平声（平），此处须填仄声字`
- `「缚」去声（仄），此处须填平声字`
- `「猿」平声（平），此处须填仄声字`
- `「子」上声（仄），此处须填平声字`

这类错误说明：  
模型虽然能组织出完整语义，但在长调强约束下，平仄控制已经明显失稳。

#### 例 2：严重字数不符

数据库：`c2`  
模型：`qwen3.6-plus`  
词牌：`沁园春`  
题目：`玄枢阁偶感`  
分数：`0`

正文片段：

> 铁律森森，云壑沉沉，剑阵列霜。叹青衿缚羽，空怀星斗；丹炉锁雾，暗减韶光……

错误：

> 字数不符：该变体要求 105 字，实际 56 字，相差 49 字

这类样本说明：  
部分失败不是“局部小错”，而是没有真正生成出完整的 `沁园春` 结构。

#### 例 3：只差 1 字也会被严格判错

数据库：`c2`  
模型：`qwen3.6-plus`  
题目：`玄渊阁规谏`  
分数：`0`

错误：

> 字数不符：该变体要求 105 字，实际 106 字，相差 1 字

这说明格律引擎对字数采用的是硬约束判定。  
这在格律研究中合理，但在模型评估中也提示我们：字数控制是模型宋词能力中的关键基础项。

#### 例 4：词牌变体智能纠偏

在 `b1/b2/b3` 中常见如下提示：

> 智能纠偏：该词字数/格律实际匹配的是「浪淘沙令」(54字)，而非「浪淘沙」

这类情况出现频率很高，例如：

- `b1 deepseek`：`36` 次
- `b1 doubao`：`36` 次
- `b2 doubao`：`34` 次

因此，这类问题应单独解释为“词牌变体匹配偏差”，不宜和严重平仄崩坏混为一谈。

### 7.1 指标含义与口径

同质化分析使用两个综合指标：

#### `core_mihi`

```text
core_mihi
= 0.55 * whole_score
+ 0.45 * sent_score
```

其中：

- `whole_score` 表示整词语义层的收缩程度
- `sent_score` 表示句级模板层的收缩程度

`core_mihi` 是不含意象层的核心同质化指标，更适合与真实宋词基线比较。

#### `mihi_full`

```text
mihi_full
= 0.45 * whole_score
+ 0.35 * sent_score
+ 0.20 * imagery_score
```

其中：

- `whole_score`：整词语义层
- `sent_score`：句级模板层
- `imagery_score`：意象与类别层

`mihi_full` 是更完整的综合同质化指标，适合解释“为什么像”。

#### 组成指标

整词层：

```text
whole_score
= 0.50 * whole_sem_p50
+ 0.30 * whole_sem_p90
+ 0.20 * whole_highsim_ratio
```

句级层：

```text
sent_score
= 0.40 * sent_highsim_ratio
+ 0.30 * sent_pseudo_ratio
+ 0.30 * top1_sent_mean
```

意象层：

```text
imagery_score
= 0.35 * imagery_jaccard_p50
+ 0.25 * imagery_jaccard_p90
+ 0.20 * top10_imagery_concentration
+ 0.10 * (1 - imagery_entropy)
+ 0.10 * (1 - category_entropy)
```

此外，为了和真实宋词比较，还使用：

```text
excess_core_homogeneity
= core_mihi - human_core_mihi
```

只要这个值为正，就表示 AI 在相同词牌范围内比真实宋词更集中。

### 7.2 严格分析与观察分析

同质化分析分成两套口径：

1. **严格主分析**
- 使用 `rhythm_score >= 80`
- 用于正式结论

2. **观察分析**
- 不设格律阈值
- 用于验证趋势是否稳健

它在 `b1 -> b2`、`b1 -> b3` 两组主题效应上的平均 `Δmihi_full` 最大（`+0.043379`），其中最显著的又是 `Δimagery = +0.111532`。

- **`doubao` 也会被主题拉向同质化，但主要是整词语义层收缩。**  
它的主题主导轴是 `whole`，说明主题首先让它在“整首词表达什么”上更相似。

- **`qwen3.6-plus` 对新增主题最不敏感。**  
它的平均 `Δmihi_full` 只有 `+0.009087`，这意味着它的很多模板化其实在基线条件里已经发生，而不是被主题显著追加出来的。

### 7.6 不同模型的收缩机制差异（whole / sent / imagery）

如果把 [neo_model_condition_profile.csv](/D:/文件/Projects/songci/result/neo-analysis/neo_model_condition_profile.csv) 与上面的敏感性表放在一起读，可以把三家主模型的“收缩机制”区分得更清楚：

| 模型 | `b1` 基线特征 | 主题加入后的主导变化 | 强约束阶段主导变化 | 可归纳的收缩机制 |
|---|---|---|---|---|
| doubao-seed-2.0-pro | `sent_score` 基线最高（`0.513098`） | `whole` 上升最明显 | `sent` 平均转正（`+0.020753`） | 先在整词语义层收缩，再在强结构条件下出现更明显的句级模板化 |
| deepseek-3.2-thinking | `b1` 基线最低，但 `imagery_score` 起点也最低（`0.064197`） | `imagery` 上升最明显 | `imagery` 仍是强约束主导轴 | 最容易把主题写成固定意象池，尤其在玉兰主题中明显 |
| qwen3.6-plus | `b1` 基线 `mihi_full` 最高（`0.506810`） | 增量最小，`sent` 变化幅度最大但为负 | `sent / imagery` 均未继续上升 | 更像是“自带模板”的模型，新增约束没有显著再推高句级收缩 |

这组比较正好回答了本项目需要补强的三个模型层问题：

1. **哪个模型更容易受到主题约束？**  
`deepseek` 最明显。

2. **哪个模型更容易在句级模板上收缩？**  
在强结构条件下，`doubao` 最明显，尤其 `b3 -> c1` 中 `delta_sent_score = +0.030238`。

3. **哪个模型更容易在意象层面收缩？**  
`deepseek` 最明显，`b3` 的 `imagery_score = 0.266538` 已经显著高于同条件的 `doubao = 0.140841` 与 `qwen = 0.175900`。

### 7.7 主题效应与强约束效应

从 [neo_condition_deltas.csv](/D:/文件/Projects/songci/result/neo-analysis/neo_condition_deltas.csv) 看，主题效应与强约束效应可以分开理解：

#### 主题效应：`b1 -> b2 / b3`

- `改革主题` 对三家模型的 `Δmihi_full` 都不大，分别是：`deepseek +0.014119`、`doubao +0.026965`、`qwen +0.006895`
- `玉兰主题` 的提升更明显，分别是：`deepseek +0.072638`、`doubao +0.040558`、`qwen +0.011278`

这说明玉兰主题通常比改革主题更容易诱发模板化，尤其是意象层模板化。

#### 强约束效应：`b3 -> c1 / b2 -> c2`

- `b3 -> c1` 中，只有 `doubao` 的 `mihi_full` 继续上升（`+0.009034`），`deepseek` 与 `qwen` 都下降
- `b2 -> c2` 中，`doubao`、`qwen` 都略降，说明 `type_e` 并不会简单地把所有模型继续推向更高同质化

因此，“更强约束 = 更高同质化”并不是机械规律；不同模型会把相同约束翻译成不同的收缩路径。

### 7.8 同质化首先来自整体语义收缩

在 `b1 -> b2` 中：

- `deepseek`：`delta_whole_score = +0.073021`，`delta_sent_score = -0.065387`
- `doubao`：`+0.085857`，`-0.045697`
- `qwen`：`+0.069881`，`-0.059620`

在 `b1 -> b3` 中：

- `deepseek`：`+0.078641`，`-0.009196`
- `doubao`：`+0.083301`，`-0.022218`

这说明：

**主题首先让模型在“整首词表达什么”上趋同，而不是立刻让每一句都更像。**

换句话说，主题约束更像是在先压缩“语义选题空间”，然后才在部分模型上进一步触发句法或意象模板。

## 8. 综合讨论

把三大分析模块放在一起看，可以得到几条更完整的结论。

### 8.1 宋词生成能力不是单一维度

一个模型可能：

- 格律更稳定，但内容更收缩
- 主题组织较丰富，但格律控制较差
- 句法较灵活，但意象系统较窄

因此，不能只用“像不像宋词”或“相似度高不高”来判断模型能力。

### 8.2 格律是最基础、最能拉开差距的门槛

在本项目中，真正最能拉开模型差异的是格律，尤其是高难长调条件下的格律稳定性。  
`doubao` 在 `c2` 中仍有 `79.76` 的平均格律分，而 `deepseek` 和 `qwen` 则分别降到 `18.59` 和 `28.68`。

### 8.3 主题与意象揭示了内容组织方式

主题统计与意象统计说明：

- `a1 / b1` 这类无主题条件并不是真正随机写作，而是已经默认回到 `离别羁旅 / 闲适隐逸` 的高频意象模板
- 玉兰主题会强烈收缩到 `咏物寄托` 和一组素洁、寒窗、春寒意象
- 改革主题会强烈收缩到 `谈禅说理 / 豪情壮志` 和宗门制度意象

更重要的是，不同模型走向这些模板的方式也不同：

- `deepseek` 更容易在意象层形成稳定主题池
- `doubao` 更容易在强结构条件下进一步转成句级模板
- `qwen` 则更像在基线阶段就已经带着较强的默认模板

这解释了为什么同质化分析中，玉兰和改革会形成不同的重复模板，也解释了为什么不同模型的“收缩机制”并不相同。

### 8.4 同质化是最终表现层，而不是唯一分析入口

同质化指标最终量化了“作品彼此有多像”，但这种相似性背后，其实依赖于：

- 词牌是否被固定
- 主题是否被收缩
- 模型能调用的意象池是否变窄
- 模型是否能稳定维持格律框架

因此在整个项目里，同质化更适合作为“最终表现层”的分析模块，而不是唯一主线。

## 9. 结论

综合本项目全部分析结果，可以得到以下结论。

1. **不同大模型的宋词生成能力确实存在显著差异。**  
在样本量充足且覆盖完整的主模型中，`doubao` 的整体格律能力显著强于 `deepseek` 和 `qwen3.6-plus`。

2. **格律是最基础的能力门槛。**  
固定 `沁园春` 的强约束条件，尤其是 `c2/type_e`，最能暴露模型在长调结构与平仄控制上的薄弱点。

3. **自由生成并不意味着真正发散。**
在 `a1` 中，不同模型会回到少数熟悉词牌，也会回到少数熟悉意象；例如 `qwen3-max` 的高频意象明显集中于 `芦花 / 西风 / 孤舟 / 烟水`，而 `doubao` 更偏 `清露 / 岸柳 / 石径 / 晚风`。

4. **主题和意象统计表明，模型的内容组织存在明显收缩。**
玉兰主题几乎压倒性地收缩到 `咏物寄托`，改革主题则稳定收缩到议论化和制度化表达；而在 `a1 / b1` 中，不同 `category` 也已经对应着几套稳定的偏好意象模板。

5. **内部同质化确实存在，但不同模型的收缩机制并不相同。**
`doubao` 的绝对同质化水平最高，`deepseek` 对主题最敏感且更易出现意象层收缩，`qwen3.6-plus` 则更像在基线阶段就已经带有较强的默认模板。  
因此，同质化不应被看作单一指标，而应理解为格律、主题、意象和结构收缩共同投射到文本相似性上的结果。

## 10. 局限与后续工作

### 10.1 样本不平衡

部分模型样本较少，例如 `gemini-3.1-pro`、`gpt-5.4-high`，不能与主模型做严格并列结论。

### 10.2 格律引擎当前采用硬约束

例如字数差 1 个字也会严格判错。  
这对格律研究合理，但在模型综合评价中，后续可以考虑引入更细粒度的容错分层。

### 10.3 意象与主题提取仍存在自动标注偏差

当前意象和主题分类足以支撑统计趋势，但在细节解释上仍可能存在噪声。

### 10.4 可视化与案例展示还有继续增强的空间

后续可以进一步补充：

- 句对证据卡片
- 整首词对照页
- 模板簇视图
- 格律错误案例页

## 11. 一句话总结

**本项目表明，大模型宋词生成能力不能只用“像不像宋词”来判断，而需要同时考察格律稳定性、主题与意象的内容组织，以及作品内部是否出现持续收缩和重复；其中格律是基础门槛，主题和意象揭示内容收缩，而同质化则是这种收缩在文本层的最终表现。**
